La machine dit au 2e proxy, via le protocole http, cad via un truc qui ressemble à un échange web banal, histoire que ça passe le 1er proxy, "stp, contacte tel serveur sur tel port et suit tel protocole pour moi, et renvoie moi, dans un paquet http, la réponse".
C'est ce qu'on appelle du IP over HTTP.
On peut imaginer aussi du SOCKS overt HTTP, du HTTP over HTTP, ...
