Comment crée une application :

Règlementation
Avant de créer votre application, merci de lire attentivement les points suivants :

les applications ont pour but de compléter ou d'améliorer le confort des jeux Muxxu, ou de fournir des outils pour la communauté Muxxu. Toute autre utilisation est exclue, il est en particulier interdit d'ajouter des jeux sous forme d'applications.
vous n'avez en aucun cas le droit de demander un mot de passe quel qu'il soit à un utilisateur se connectant sur votre application
vous n'avez en aucun cas le droit de demander un paiement, un don, ou quoi que ce soit d'autre pour accéder à tout ou partie de votre application
vous n'avez pas le droit d'utiliser des scripts de récupération régulière de données. Seules la visite ou les actions d'un utilisateur sur votre site vous autorisent à récupérer des données à ce moment précis
Motion-Twin, société créatrice de Muxxu, se réserve le droit de ne pas activer et de désactiver les applications de son choix
Pour toute question relative aux applications, merci de nous contacter via le Support. Attention ! nous ne pourrons répondre à aucune question concernant la programmation des applications, merci de contacter d'autres créateurs d'applications si vous désirez une aide technique.

Documentation
L'ensemble des données exportées par Muxxu et les jeux Muxxu sont au format XML. Les données sont séparées en deux catégories :

les données publiques : elles sont librement accessible par votre application
les données privées : elles nécessitent le consentement de l'utilisateur avant que votre application puisse y accéder
Chaque donnée a un identifiant et nécessite une clé d'accés.

Pour accéder à des données, il faut vous faire un appel HTTP sur l'adresse http://muxxu.com/app/xml en passant les paramètres suivants :

app : votre identifiant d'application
xml : le type de données à laquelle vous voulez accéder
id : l'identifiant des données demandées
key : la clé d'accés, calculée à partir de md5(priv-key . data-key)
md5 est une fonction cryptographique accessible depuis la plupart des langages de programmation
priv-key est la clé privée de votre application, que vous obtenez une fois votre application créée
data-key est la clé d'accés aux données demandées
. veut dire que l'on concatène les deux chaînes de caractères ensemble
act (optionel) : l'action qui a été déclenchée sur votre application
Par exemple, un appel pour récupérer les données de l'utilisateur 578 ressemblera à cela :

http://muxxu.com/app/xml?app=myapp&x ml=user&id=578&key=a1e896e52b4a0 e13f464c2a6e54ed590
Informations Utilisateur
Lorsequ'un utilisateur se connecte à votre application, Muxxu vous transmet les données suivantes via les paramètres GET :

uid : l'identifiant Muxxu de l'utilisateur
name : le pseudo de l'utilisateur
pubkey : la clé permetttant l'accés aux données publiques user de l'utilisateur
Si ces données ne sont pas présentes, cela veut dire que l'utilisateur n'est pas connecté à Muxxu.

Données publiques
Les données publiques peuvent être accédées via la clé de données correspondante. Par exemple pour accéder aux données publiques de Kingdom, il vous faudra d'abord obtenir les données publiques Muxxu de l'utilisateur, qui contiendront la clé permettant d'accéder aux données publiques Kingdom.

Données privées
Les données privées nécessitent quant à elles une autorisation explicite de l'utilisateur. En effet si vous tentez d'accéder à des données privées alors que l'utilisateur ne vous a pas donné ce droit, vous recevrez le message d'erreur suivant :

<needauth/>
Vous devez donc rediriger l'utilisateur via l'adresse web http://muxxu.com/app/auth en passant les paramètres suivants :

app : votre identifiant d'application
xml : le type de données à laquelle vous voulez accéder
id : l'identifiant des données demandées
key : la clé d'accés, calculée à partir de md5(priv-key . xml . id . 'AUTH')
md5 est une fonction cryptographique accessible depuis la plupart des langages de programmation
priv-key est la clé privée de votre application, que vous obtenez une fois votre application créée
xml le type de données à laquelle vous voulez accéder
id l'identifiant des données demandées
'AUTH' la chaîne de caractères constante
act (optionel) : l'action a effectuer sur votre application une fois l'autorisation approuvée
Une fois que l'utilisateur a approuvé l'accés aux données privées, vous pouvez utiliser la clé de données xml . id . 'GET' pour pouvoir y accéder de la même façon que les données publiques.

Liste des données
Vous pouvez consulter la liste complète des données publiques et privées auxquelles vous pouvez accéder pour chaque jeu.

Et maintenant ?
Vous pouvez si vous le désirez remplir le formulaire pour créer un application ici.

Votre application sera désactivée par défaut, elle ne sera donc pas accessible par d'autres joueurs que vous-même. Une fois que vous avez fini de développer votre application, merci de contacter le Support en nous indiquant l'utilisation de votre application.

Motion-Twin se réserve le droit de ne pas activer ou de désactiver ultérieurement votre application.
