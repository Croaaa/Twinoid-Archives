Suggestion : Changer le code pour le smiley .

Quand on écrit un message avec un nombre entre parenthèses et que ce nombre se termine par 8, la combinaison "8" + ")" est automatiquement transformée en alors que ce n'est probablement pas ce qu'on voulait.

Il m'est déjà arrivé assez souvent de copier-coller du texte contenant des nombres entre parenthèses et d’être surpris par le résultat. Et cette erreur se voit assez régulièrement sur les forums.

Ce problème se manifeste aussi lorsqu'on inclut un ID de joueur. Par exemple, si on utilise le bouton Joueur ou qu'on tape directement "@Bugzilla:148", on obtient :
Bugzilla Mais si on écrit exactement la même chose entre parenthèses "(@Bugzilla:148)", alors on obtient un résultat totalement différent :
(epoq Et non, epoq n'est pas mon identité secrète...

Si on voit l'erreur avant de poster, on peut dans certains cas contourner le problème en remplaçant "8)" par "8----)", "8[i][/i])" ou une autre balise vide. Mais ce truc ne fonctionne pas dans les sondages. Si on veut inclure un nombre entre parenthèses dans une question de sondage, le truc des balises vides ne fonctionne pas car les balises de mise en forme ne sont pas interprétées alors que les smileys le sont. La seule solution est d'inclure un espace, un point ou un autre caractère visible (ou un caractère Unicode invisible ou presque, mais peu pratique).

D’où ma suggestion : changer le code de ce smiley pour qu'il ne cause plus de problèmes.

Nouveau code possible pour :

  • "8-)" (ajout d'un tiret)
  • ou bien conserver "8)" mais ne faire la conversion que si c'est précédé d'un espace : " 8)".
