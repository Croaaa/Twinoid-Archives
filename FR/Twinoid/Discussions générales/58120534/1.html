Pour les victimes de ce bug : le problème vient d'un changement effectué par Facebook avant-hier. Ce n'est pas vraiment un bug Twinoid, mais une conséquence d'un changement qui avait été annoncé par Facebook il y a quelques mois et qui vient d’être effectué, empêchant les anciennes applications d'utiliser l'identification Facebook.

Pour les détails techniques :

  • Le système d'identification proposé par Facebook existe en plusieurs versions numérotées : 2.1, 2.2, 2.3, ..., 2.8. Twinoid utilise la version 2.2 depuis deux ans. Or depuis avant-hier, Facebook a supprimé la version 2.2 et oblige les développeurs à utiliser au minimum la version 2.3 (qui sera à son tour supprimée en juillet) ou une version plus récente.
  • Le problème est qu'entre la version 2.2 et la version 2.3 ou les suivantes, Facebook a changé l'encodage utilisé pour les données d'identification. Donc cela demande aux admins Twinoid d’écrire du nouveau code qui puisse décoder le nouveau format de données utilisé par Facebook pour identifier les utilisateurs.

Cela explique que ce ne soit pas une correction à faire en cinq minutes car il faut écrire du nouveau code du côté Twinoid. Le changement à faire dans le code n'est pas très complexe d’après ce que j'ai pu voir, mais il faut qu'un admin trouve le temps de le faire.
