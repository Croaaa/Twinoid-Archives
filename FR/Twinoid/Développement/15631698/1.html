La syntaxe est assez intuitive pour quelqu'un qui a déjà programmé, c'est déjà un bon point (toujours les accolades, les fonctions et tout).

Pour quelqu'un qui n'a jamais appris, ce langage pourrait se relever intéressant car il propose une syntaxe simple sans superflux:

  • en PHP par exemple il y a des dollars devant les variables, dans notre langage non, bon ok c'est pas grand chose, mais ça simplifie la lecture.
  • en C le typage est statique, dans notre langage c'est au choix du développeur, pour les débutants, ça peut être un bon point pour commencer plus facilement.
  • les fonctions ont un nom qui précise bien le type que l'on manipule, par exemple pour récupérer le nombre d'éléments dans un array il faut faire Array.size(tableau). Il faut aussi savoir que rien est objet, le "Array." n'est là que pour faciliter la compréhension, il est assez facile de ce souvenir du nom des fonctions rapidement.

La fonction File.countLine() telle qu'elle est implémentée dans la VM lit le fichier octet par octet jusqu'à la fin du fichier, et ajoute 1 à la variable retournée à chaque fois qu'elle rencontre un '\n'. Donc aucun malloc n'est effectué, la consommation de ressource est nulle.

cmp condition,lbl_si_vrai,lbl_si_faux

La boucle while

lbl_start:
cmp Condition,lbl_cont,lbl_end
lbl_cont:
 // Instructions dans la boucle.

lbl_end:
 //Instructions après la boucle

La boucle do-while
Elle est modélisée par :

lbl_start:
 // Instructions dans la boucle.
cmp Condition,lbl_start,lbl_end
lbl_end:
 //Instructions après la boucle
