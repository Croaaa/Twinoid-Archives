Passer par un calcul dans un autre langage alors que tu as entre tes mains un langage plus ou moins puissant selon ton SGBD ? Ah.

Revenons à la question initiale :

SELECT * FROM table WHERE points > views

Possible si points et views se trouvent dans table.
Si views est une variable, eh bien ... Tu sais quoi faire.

UPDATE table SET champs3 = champs1 - champs 2

Bien, si champs1, champs2 et champs3 appartiennent à table, mais si ce n'est pas le cas vous avez les jointures. Mais attention, le résultat se répercute sur TOUTE la table.
Pourquoi c'est mieux ? Parce qu'il n'y a pas à faire d'autre requête de sélection derrière.

En plus, ce qui est marrant, c'est que même si vous vouliez faire comme ça, vous n'auriez même pas à faire vous-même la soustraction, SQL gère ça !

SELECT A - B
FROM T

Retourne tous les résultats des différences des champs A par B sur la table T.
