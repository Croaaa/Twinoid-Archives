Ce n'est pas vraiment un bug si cet event handler s’exécute dans le contexte de la page. GM_xmlhttpRequest ne peut s’exécuter de manière sûre que dans le contexte GM.

Il me semble que la solution correcte à ce problème est d'utiliser un timer ou de passer des messages entre une fonction qui s’exécute dans le contexte de la page (event handler) et une autre qui s’exécute dans la sandbox GM (via timer ou autre type de trigger qui n'utilise pas unsafeWindow). Ce sera forcément asynchrone, mais cela évite d'introduire des failles de sécurité en tentant d’exécuter du code GM dans le contexte de la page.

Voir aussi cette page qui explique le problème et des solutions possibles.
