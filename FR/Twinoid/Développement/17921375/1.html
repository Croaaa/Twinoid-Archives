L'idée serait que nous, utilisateurs, ajoutions le son qui manque tant aux jeux MT.

Pas juste une bête playlist externe que chacun peut faire soi-même, mais un système intégré et qui s'adapte.

  • Un module pour se greffer à la page et jouer le son.
    Probablement Greasemonkey + Flash ou Java.
  • Un module pour repérer ce qui est en cours pour envoyer le son approprié.
    Probablement Greasemonkey pour analyser le html.
    Et là où ça ne suffit pas, du Flash ou Java en complément pour analyser le rendu. Il me semble que c'est possible, et rien que la dominante de couleur peut être très informative.
  • Un moyen de compiler les sons à jouer.
    Ça peut aller d'un simple sujet forum pour proposer avec les développeurs qui choisissent, à une interface utilisateur en ligne pour proposer et étiqueter avec équipe de tri/vérification/modération complète derrière.

Vu que ce serait peu réactifs, ce serait à priori que des musiques d'ambiance et non des effets accompagnant les actions.

Que des musiques libres d'utilisation bien sûr, heureusement il y en a.

Comment vous faites pour mettre à jour votre script en même temps que l'ajax de Hordes ?
Le 5 novembre 2012
Tyler_Durden
Voleur de fichiers

Tu modifies le prototype de l'objet XMLHttpRequest directement si tu veux que ton script se déclenche en même temps/au callback de l'ajax de hordes.

Perso, voici le hook que j'utilise :

var send = window.XMLHttpRequest.prototype.send,
 onReadyStateChange;
function sendReplacement(data) {
 if(this.onreadystatechange) {
 this._onreadystatechange = this.onreadystatechange;
 }
 this.onreadystatechange = onReadyStateChangeReplacement;
 return send.apply(this, arguments);
}
function onReadyStateChangeReplacement() {
 if(this._onreadystatechange) {
 var temp = this._onreadystatechange.apply(this, arguments);
 if (this.readyState == 4) {
 // exécution de ta fonction ici, au callback ajax
 }
 return temp;
 }
}
window.XMLHttpRequest.prototype.send = sendReplacement;

EDIT : Attention à ne le faire qu'une fois par page. Si cette portion de code est exécutée régulièrement, tu vas atteindre un niveau de récursivité trop élevé et faire planter toute requête AJAX sur la page.
