LESS c'est comme du CSS, en mieux.

Déjà tu as des variables:
@var: #FFFFFF;

Tu as des fonctions:
.fonction {
.... -mos-border-radius: 3px;
.... -o-border-radius: 3px;
}

#id {
.... .fonction; // retourne ce qui est contenu dans la classe .fonction
}

Tu as aussi des imbrications: (& fait ici référence à a)
a {
.... background-color: blue;
.... &:hover {
.... .... background-color: red;
.... }
}
Retournera:
a { background-color: blue; }
a:hover { background-color: red; }

Puis tu as des fonctions toutes prêtes pour assombrir une couleur, l'éclaircir et tout, tu peux aussi faire des calculs (ex: 2px + 2px = 4px)

[2403]

Le plus simple à faire est un compilateur pour une VM que tu crées en même temps.
Par exemple tu as ton programme :

Programmesay[%"Je suis un Dieu (et modeste aussi)."].
end 5.Si on considère que say est défini par l'opcode 0x85 immédiatement suivi de l'adresse d'une chaine de caractère sur 4 octets en Big Endian (pas d'inversion d'octets sur les adresses), que end est défini par l'opcode 0xFF immédiatement suivi de la valeur de retour du programme sur un octet, et que toutes les chaînes sont terminées par 0x00, alors le compilateur donnera ça en sortie (hexdump) :
Programme compilésay=85 adresseChaine=00000007 end=FF valeurRetour=05 Chaine=4a65207375697320756e204469657520286574206d6f6465737465206175737369292e00(J'ai mis les correspondances pour clarifier, mais elle ne figurent pas dans le fichier binaire)

Et la VM lira ce fichier binaire, repérera l'instruction "say" (0x85), lira les 4 octets suivants pour déterminer l'adresse de la chaine (0x00000007), affichera les caractères à l'adresse 0x07 jusqu'à ce qu'elle rencontre 0x00, puis elle repèrera l'instruction "end" (0xFF), et renverra l'octet suivant (0x05) au système d'exploitation comme valeur de retour du programme.
Si tu veux je t'implémente un mini-compilateur/VM qui permet d'effectuer quelque instructions basiques
