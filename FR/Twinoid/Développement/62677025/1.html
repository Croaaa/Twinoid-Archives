TL;DR: Créer un serveur qui fera l'interface entre ton front et l'api de Twinoid

J'arrive peut-être un peu tard mais c'est typique des problèmes de CORS en front.

CORS est une solution qui empêche le javascript d'un domaine d'interférer avec un autre domaine, histoire d'éviter des petits imprévus du genre le vol de cookies. Il est incorporé dans tout les navigateurs depuis IE je sais plus combien.

La seule façon de corriger ce type d'erreur est de créer un serveur (ou utiliser une solution serverless comme AWS Lambda par exemple) qui s'occupe de réaliser les requêtes vers l'api Twinoid.

Ton serveur enverra ensuite au front les données récupérées, il n'y aura pas de problèmes de CORS du coup puisque c'est toi qui gère ça de ton côté
