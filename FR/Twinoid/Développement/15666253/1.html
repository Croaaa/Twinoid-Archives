javascript et PHP sont deux mondes très différents. L'un est tourné vers le client, comme HTML et CSS, l'autre vers le serveur.

Après, on peut faire des applications de bureau en PHP ou bien gérer la partie serveur d'un site web en javascript (node.js) Il faut peser le pour et le contre. Faire le server-side en js permet d'avoir un seul langage pour tout le site, mais le PHP est offre un plus large choix de serveur.

Après, il y a tout un paquet d'alternative. Java (patapé) est le second langage utilisé pour le web et peut-être le premier dans le domaine industriel. python est aussi un choix populaire dans certain milieux, avec par exemple django. D'autres solitions plus exotiques existes tel que ruby on rail, nitrogen... Je pense que wikipedia sera plus exhaustif que moi.

______
Petit coup de pub pour la route, appengine, de google, m'a permis d'expérimenter le python server side et a le bon gout de proposer une offre gratuite correcte. J'utilise actuellement node.js en étant hébergé chez cloudno.de. Pour le moment, c'est en beta et gratuit.
Après, tant qu'il n'y a que du html/css, DropBox est une solution d'hébergement génial. Le lien ci-dessus est sans parrainage, avoir un parrain permet d'avoir plus de place gratuitement.
