- Ensuite, pour ce qui est de pouvoir compiler d'un langage à un autre. Pour illustrer, je vais prendre Teacher Story comme exemple.
Le jeu a 4 composants:

  • un client flash (pour l'interface utilisateur)
  • un serveur web (pour la persistence des données et les intéractions online)
  • du Javascript pour le dynamisme des pages HTML
  • un "solveur", qui contrôle le coeur du jeu.

Tout est écrit en haxe et ça plusieurs avantages conséquents:

  • même si l'API change (c'est à dire les fonctions spécifiques à une plate-forme, comme le MovieClip pour Flash), le fait d'avoir 1 langage permet à un dev de mettre les mains hyper facilement autant dans le code du client Flash que celui du serveur Web, ou du JS.
  • le solveur est écrit entièrement sans aucun appel spécifique aux API. En clair, le même code est capable de tourner indifféremment sous Flash, web ou même JS (si on en avait besoin). Ce solveur s'attèle uniquement à la logique du jeu. Le prof par exemple a des stats (point de vie, XP, listes de sorts...) mais aucune information graphique. Le solveur est capable d'initialiser les données d'un cours, il peut recevoir en entrée une action du prof et renvoie en réponse le résultat de cette action puis les actions des élèves.

Ce solveur est donc totalement cross-platform car il ne contient que des choses simples (tableaux, boucles, fonctions, enum, ...).
Dans le cas de TS, le jeu tourne entièrement côté Flash, mais la même partie est répliquée côté serveur pour vérifier l'intégrité des actions (pas de hack possible).

Je rajouterai que dans le cadre de compilation d'une même application/jeu sur plusieurs plateformes (android, iOS et flash par exemple) plutôt que d'un jeu unique mais qui s'étend sur plusieurs plateformes (comme Twinoid dont deepnight a expliqué le fonctionnement), Haxe a également un gros intérêt :

Si l'application/jeu a été développée en suivant quelques règles, le même code source peut être compilé vers ces différentes plateformes tel quel. Seule la ligne de commande pour compiler change. Après, il faut développer l'application/jeu en ayant en tête l'export vers ces plateformes pour faire en sorte d'avoir les meilleures performances sur chacune d'entre elles (un jeu qui tourne très bien sur flash peut être très lent sur android suivant comment il est codé, par exemple).

Pour pouvoir faire ce genre de multi-plateforme efficacement avec Haxe, l'utilisation d'OpenFL (anciennement NME) est quasi-indispensable. OpenFL permet entre autres d'éviter d'avoir à faire les imports conditionnels dont tu as parlé
