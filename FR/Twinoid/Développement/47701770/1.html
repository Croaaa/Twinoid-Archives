Que ce soit pour CSS ou Javascript, une bonne pratique consiste à avoir tout le code source dans un dépôt local git ou autre, à faire des tests locaux sur cette machine, et avoir un script qui "publie" les résultats sur un serveur public externe. Ce script peut être invoqué lors d'un "git push" ou bien il peut s'agir de "Makefile", mais c'est une autre histoire. Ce script d'envoi au serveur externe peut se charger de la minification du code, de la compression et éventuellement de l'obfuscation.

L’idée de base est qu'on ne travaille que sur la version d'origine du code et donc il n'est jamais nécessaire de "dé-minimiser" le code. Les tests initiaux pendant le développement se font aussi sur cette version non compressée.

Les étapes de minification et compression appartiennent plutôt aux outils de publication, pas à l’éditeur. Il me semble que cette demande de dé-minimiser dans l’éditeur n'est pas une bonne idée car cela implique qu'on utilise une méthode de travail qui n'est pas vraiment la bonne.
