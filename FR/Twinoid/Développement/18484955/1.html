Screen

Screen


Il s'agit donc en résumé d'un simple script qui ne fait qu'ajouter un lien et afficher la rentabilité d'un lieu dans une nouvelle colonne.
Voici donc les utilités de ce script un brin utile .

Récupérer les positions des généraux ennemis/alliés : pas possible à cause du flash comme tu le disais Flagoul.

Importer le graphique de kingdomSim est plus compliqué qu'il n'y parait. Il est en flash d'une part, et d'autre part, il semble qu'il fait un check du domaine sur lequel il est utilisé (à première vue).

Par contre, j'ai creusé un peu tout ça, et je te donne le gral : les coordonnées des points sur le graphique.

Voici donc très précisément comment récupérer les données mises à disposition par kingdom :

http://www.smoggstudio.com/kingdom/data.php?commerce=[COMMERCE]&garnison=[GARNISON]

Un exemple. Le code retourné est en JSON, et la partie qui nous intéresse est retour.elements[0].values
C'est un array javascript dont chaque élément y, en position x de l'array (x commençant à 0), est la valeur du commerce (y) au bout de x tours joués.

A partir de cet array, tu peux tout faire. Construire le graphe, fixer un "objectif commerce" sur tes territoires individuellement et afficher un "compteur de tours à jouer" dans la page capitale en fonction, ...

Bon, le soucis, c'est que kingdomSim n'accepte pas le cross domaine. Mais comme je suis un mec sympa, je t'ai fait un petit script PHP sur mon serveur qui te transfère les données de kingdomSim (via cURL) et accepte le cross domaine.

Je t'invite donc à utiliser cette url :

http://guizmus.fr/kingdom.php?c=0&g=100

(dans cet exemple, "c" est le commerce actuel à 0, et il y a une garnison "g" de 100).

De plus, comme j'avais pas envie de m'arrêter là, voici la fonction jquery à utiliser pour pouvoir directement appeler ton callback personnel sur l'array décrit précédemment :

var kingdom_sim = function (callback,commerce,garnison) {
 $.ajax({
 url : "http://www.guizmus.fr/kingdom.php?c="+commerce+"&g="+garnison,
 success : function(ret) {
 try {
 eval("var retour = "+ret);
 callback(retour.elements[0].values)
 } catch (e) {
 console.log("erreur dans la récupération des informations : ",e);
 }
 }
 })
}

Enjoy

Concernant les fonctions "a moitié déclarée", tu aurais du demander au lieu de bloquer ^^
Utilise window.nom_de_ta_fonction pour être certain que cette fonction soit bien déclarée sur window, et non sur l'espace de travail des extensions.

Pour explication, les extensions chromes ont toutes leur "onglet invisible" dans lequel elles tournent, leur permettant d'avoir des données persistantes et indépendant de l'instance/l'onglet dans lequel elles s'exécutent visuellement.
Déclarer une fonction classiquement (var x = function (machin) {truc}) les déclare dans cet espace. Dès lors, quand tu les appelles via un onclick, le contexte étant l'objet window de l'onglet d'exécution, la fonction n'est pas trouvée.

(le paragraphe qui suit concerne la compatibilité avec firefox. N'en tiens pas compte si tu n'as pas encore réfléchit à ce point là.)
Pour assurer une plus grande généricité, et pouvoir plus tard proposer ton script sur firefox/GM, je t'invite à utiliser "unsafeWindow", objet que tu déclares en début de script comme étant égal à window dans le cas où unsafeWindow n'est pas déclaré.
unsafeWindow est l'objet window classique, celui de la page web en elle même, mais est utilisé sous cet alias par GreaseMonkey pour bien signaler à l'utilisateur qu'il est en train d'accéder à la page web elle même, et donc qu'il y a un risque concernant les données qu'il trouvera.

Tu peux faire une alternative à ce calcul pour la décroissance. La formule mathématique reste la même, elle est juste inversée. Au lieu de tendre à 2*G (G étant la garnison), elle va tendre à -E², E étant l'éloignement à la capitale.

En supposant que tu connais E, tu obtiendras la courbe voulue en traçant la courbe basée sur l'opposée de la simulation de paramètres :
- Commerce de départ : -commerce actuel
- garnison : E²/2

Pour ce qui est des grands nombres, c'est normal qu'il ne marche pas. Comme tu as vu, l'array des valeurs retournées comporte une case par X, et non l'inverse de la fonction (1 X par nouvelle valeur de commence). La longueur de l'array est donc exponentielle, et je pense que ton script pourra mathématiquement ramer passé certaines valeurs (recevoir un array d'un million d'élément est rarement apprécié par javascript)

  • Le include ligne 5 n'est pas interprété par chrome.
