Bonsoir,

Je vais compléter un peu ce qu'a dit Liezon.

Pour développer une application web, je peux te conseiller plusieurs stacks selon tes besoins et ton niveau en développement. Pour commencer, si tu débutes en programmation, je te conseille Python, qui est un langage assez simple à comprendre et qui te permettra de progresser assez rapidement. Il faudra aussi que tu regardes du côté du C++/C# pour comprendre le concepts et les paradigmes de la programmation orienté objet.

Après ça, je te conseille cette stack pour commencer le développement web :

Nginx/Apache + PHP5 + MySQL + JS (jQuery..etc)

Pour bien organiser ton code, il faut absolument que tu utilises un framework comme cakePHP, Laravel, Symfony avec une architecture MVC. Oublie le code procédurale/spaghetti/on réinvente la roue/fait maison... c'est horrible pour ceux qui bossent avec toi, c'est pas pro du tout et ce n'est absolument pas maintenable à moyen terme.

Une fois que tu as développé quelques petites applications et que tu as acquis assez d'expérience, tu peux essayer d'autres technos comme Ruby avec cette stack qui est assez mature pour le monde pro :

Nginx (avec Passenger) + Ruby on Rails + Backbone + MongoDB

Nginx en tant que serveur web c'est vraiment nickel, la configuration est simple et il est performant et bien optimisé, rien à redire.
BackboneJS pour structurer l'application avec le modèle MVC.

Je te propose une dernière stack, qu'on est en train d'utiliser avec Liezon pour le développement de KohLambda (app pour hordes.fr) :

Heroku + NodeJS + ExpressJS + PostgreSQL + redis + socket.io

Heroku : Paas pour l'hébergement et le maintient de la plateforme exécutant notre application.
NodeJS : Langage de programmation asynchrone (Javascript côté serveur)
ExpressJS : Le framework web
PostgreSQL : Système de gestion de base de données (équivalent à MySQL)
Redis : BDD NoSQL (non relationnel)
socket.io : Pour les apps temps réel

Par contre, tout ce que je t'ai dit n'est pas suffisant un jeu web de grande envergure. Dans ce cas, il faudra que tu décomposes ton projet en modules/composants indépendants. Par exemple pour un jeu web tu peux avoir ceci :

- Un module pour de gestion des parties/affrontements entres joueurs/IA
- Un module pour la gestion des scores et des classements
- Un module pour les récompenses/distinctions/succès/trophés...etc
- Un module pour le forum
- Un module pour le tchat
...

Tous ces modules doivent être indépendants et communiquer entre eux grâce à un système de messagerie AMQP (RabbitMQ, IronMQ...etc) fonctionnant avec un système de workers/queues, c'est à dire que chaque module (worker) écoute sur une ou plusieurs queue(s) et effectue un traitement à chaque fois qu'un message arrive.

Ce type d'architecture (décomposition en module) est beaucoup plus pratique et plus souple que d'avoir UNE application web qui fait tout (interface web + gestion des parties / des classements / des joueurs / des forums...) c'est trop lent et lourd gérer avec des requêtes HTTP (problèmes d'accès concurrents...etc).

Tu dois te dire que si tu as une requête vers ton application web qui prend plus de 1/2 secondes pour s'exécuter, tu peux envisager d'utiliser un service en arrière plan (background jobs) pour traiter les informations.

J'espère que ça t'aidera à comprendre comment concevoir une application web d'une manière efficace et organisée. Si tu as des questions n'hésite pas.

Bonne soirée.

Les langages les plus utilisés dépendent fort du domaine dans lequel on travaille. Pour ma part, je travaille dans le domaine des télécoms, principalement sur des services en réseau. Si je trie en fonction de la quantité de code écrit, je crois que les langages que j'ai utilisés le plus souvent sont le C, Perl, Java, Python et un peu de C++, Erlang et d'autres langages. Par exemple, pour des applications réseau (client-serveur ou proxy), j'utilise :

  • Java + Netty
  • Perl + POE
  • Python + Twisted
  • Javascript + jQuery
  • ...

Il y a aussi d'autres bibliothèques utiles pour créer des interfaces utilisateur (p.ex. GTK+ pour le C), pour gérer des bases de données, et des tas d'autres pour différents domaines.

Il est important de connaître les outils disponibles et d'utiliser autant que possible le meilleur outil pour chaque type de boulot. On peut essayer de résoudre tous les problèmes avec un seul outil (langage de programmation) mais ce n'est pas toujours le meilleur choix.

Quels sont les language de programation que vous utilisé pour faire les jeux comme horde ou autre

Motion Twin utilise le même langage sur les serveurs que pour le côté client (javascript généré + flash) : Haxe.

Pour le côté serveur, ils utilisent Neko VM qui tourne sur un serveur Tora communiquant avec Apache. Tora permet d'avoir des applications web persistantes qui stockent une bonne partie des données initialisées en RAM pour que 99% des chargements soient extrêmement rapides.

Tous ces outils (sauf apache..) ont été créés par warp, qui n'est maintenant plus chez MT mais continue de travailler sur Haxe & co.

Un gros avantage de Haxe pour quelqu'un qui débute en dev' web, c'est qu'il suffit d'apprendre un seul langage de programmation. Pas besoin de javascript, on le génère avec Haxe (même si perso je préfère continuer à l'écrire à la main). Les fichiers flash sont aussi générés par Haxe. Reste HTML & CSS, qui ne sont pas des langages de programmation..

L'inconvénient par contre, c'est que même si le langage en lui même est tout à fait abordable du point de vue d'un développeur (même de façon objective), le soucis est le manque de documentation qui rend l'apprentissage plus compliqué pour les débutants / développeurs formés au SDZ / etc. Il faut savoir se débrouiller avec une API.

warp s'est attaqué à ce point faible, mais c'est encore insuffisant pour les accros aux tutos & co. La mise en place de l'environnement (haxe 3 + neko 2 + tora) peut être un peu chiante, surtout qu'au moindre problème on ne peut pas compter sur notre ami google

En lisant ce topic, je pense qu'on inonde l'auteur du post de configurations plus exotiques les unes que les autres pour lui, qui a l'air de commencer dans le domaine web 
Je retiens deux possibilités pour débuter le développement web; du côté multi-utilisateurs :
 La méthode Old-School
Installer Wamp, et suivre le tuto du site du zéro. 
Le combo Apache / Php / Mysql + JS/HTML/CSS  permet un démarrage très rapide et la création d'applications simples à très complexes.
Toutefois, Apache, PHP et MySql sont très en retard sur ce qu'il se fait de neuf sur la toile, notamment vis à vis de l'optimisation de leur fonctionement et du support de la charge.
 La méthode "nouvelles technos"
Apprendre le javascript et s'éclater avec Node.js
Là encore, HTML et CSS sont nécessaires; mais c'est assez simple à assimiler.
L'intérêt de node.js est que le serveur est codé avec le (quasi) même langage que le client.
Pour faire des applications temps réel ou des jeux multijoueurs; c'est l'idéal.
Après, si le besoin se fait sentir, il peut être intéressant de coupler cela au framework Express & à une base Redis, mais ça dépend vraiment de chaque application.
Concernant la création de jeux vidéos...
Il suffit de voir le Ludum Dare pour comprendre qu'il n'y a pas de langage idéal pour faire des jeux. Flash est assez bon dans le domaine, mais très propriétaire.
Pour les fainéants, Unity permet de faire rapidement des bons jeux.
Bien sûr, l'incontournable Java, avec éventuellement des librairies (type LibGdx) pour exporter le jeu aussi bien sous PC, Mac, Linux, Android, ou même HTML5.
Pour les courageux, les bas-niveaux C,C++,C# sont également de la partie, coûplés aux librairies qui vont bien. Mais c'est clairement plus difficile 
Il en existe pleins d'autres ! Je n'ai parlé ici que de ceux que j'ai testé ! 
Concernant les Framework
C'est utile pour structurer un projet fait un équipe, avant toute chose.
Grâce au framework, les développeurs doivent suivre des contraintes fixées pour développer correctement. Pas pour rien que c'est un Cadre de travail.
Je conseille CodeIgniter pour PHP. L'appli "Miss'n mister hordes" sur l'annuaire Hordes fonctionne d'ailleurs grâce à ce framework. Assez simple à prendre en main avec le bon tuto, et disponible pour un mutualisé (corriger si besoin pour l'url rewriting).
Un framework doit être choisi correctement. Il s'agit de dimensionner son choix en fonction de son besoin : installer symphony sur un site ayant pour seule vocation d'afficher quelques pages d'information, c'est surdimensionner. Installer une petite librairie/framework perso sur un site bancaire, c'est... de la folie.
Concernant les écoles
Pleins de possibilités là encore... Prépa puis école d'ingé, école d'ingé post bac (INSA, EPITA...), écoles d'expertise (EPITECH, SUPINFO...), IUT spécialisé...
Il faut se renseigner  La plupart de ces écoles s'intègrent sur concours.
Je peux renseigner sur l'INSA (Rennes) si certains se posent des questions, étant en deuxième année 
Concernant le métier
Le diplôme joue énormément, mais les expériences, la personnalité, et les compétences permettent d'accéder à différents postes, et donc différents salaire.
Oh, chose importante : si tu veux bosser dans l'info uniquement pour gagner plus de sous, c'est pas terrible. Mieux vaut être motivé par autre chose que l'argent dans ce domaine.
