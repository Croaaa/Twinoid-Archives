Continuons un peu le hors-sujet technique...

Utiliser "content: url(...)" dans le CSS fonctionne avec Chrome et Safari, mais pas avec Firefox. Et dans ce cas c'est malheureusement Firefox qui a raison, car selon HTML5 la prioriété "content" ne devrait fonctionner qu'avec les pseudo-sélecteurs "::before" et "::after", pas pour un élément normal.

Solution : au lieu d'utiliser "content: url(...)", on le met en fond avec le truc suivant :

@-moz-document domain("www.hordes.fr"), domain("www.die2nite.com"), domain("twinoid.com") {

 img[src$="/r_ginfec.gif"] {
 width: 0 !important;
 height: 0 !important;
 padding: 16px 17px 0 0 !important;
 background-image: url("data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAABAAAAARCAYAAADUryzEAAAABGdBTUEAANbY1E9YMgAAABl0RVh0U29mdHdhcmUAQWRvYmUgSW1hZ2VSZWFkeXHJZTwAAAFhSURBVHjajFSxTsNADL1DKFnYGPIBhW/IlOxQkSn5gg7AkilDxcJWFSlTviPqEFQxsSRTdhaE+gHZWDMdeS6+XuilwtJJPvs9n+3XVCqlxJRJKSk5YOQU5twWdF2XiG/b7eje9/1RoTMbebO4Er7v6xh8xLjQZAdMfvm4POoKsc1ijzE7kbwDk/y0XFrnXa3X4vbiUzy/fx/G+S2ggiCgM8yNAJ2iKOjwHTnG7anqsIMkSejlr92O7lF0p19mHzlggD2pAuzx/kH717OZqKpXu9RoA3oPbVGA57+Zz0dAlhR7gDVNQ78P3UHbtlo6kOM4FmEY0r2ua4pxEWBPquB5HpHTNKXcsEgq0nXdtAo4juOo4WW9efg4rAh8YBhPKmB+tItX/mvAggPuSIU8z0WWZRr0d4SqqghjVcFcGopEUTRaoknGvSxLUkEvkYuwgWAaCrIxeaSC+f3b/gOmcj8CDABmEO4W5h8dKQAAAABJRU5ErkJggg==") !important;

... et cela devrait fonctionner dans tous les navigateurs car c'est conforme au standard HTML5.
