Je pense en effet qu'il y a un "bug" et que ce n'est pas voulu.

La balise {rnduser } a été crée pour remettre en place la balise {habitant } avec l'idée qu'elle pourrait servir à terme dans d'autres cas sur Twinoid. (D'où le fait qu'elle fonctionne avec les Ids Twinoid).

La balise {habitant } appelle donc la balise {rnduser:TousLesIdsDesJoueursVivants).

Le fonctionnement de Twinoid, fait que cette balise {rnduser } est forcément visible dans les gadgets RP ce qui en devait pas être le cas initialement (D'où l'absence d'icône).
Et d'où le fait qu'elle manque d'explications sur son fonctionnement.

Il aurait fallu de la part des admins faire en sorte que la balise {rnduser } est un affichage particulier.
Et que lorsqu'on appelle la balise {Habitant }, cette dernière utilise la balise {rnduser } mais en ajoutant une classe CSS supplémentaire pour lui donner une autre apparence.

Ce qui permettrait de différencier la balise {habitant } de la balise {rnduser } et éviter toute possibilité de triche.

On remarque bien que les autres balises RP ont un "rappel" dans leur affichage sur les forums qui indique quelle balise on a utilisé.
