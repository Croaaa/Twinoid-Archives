Bonjour,

C'était le cas avant la S8 à cause d'un bug dit "d'inversion de défense".

La liste des veilleurs respecte les ID des joueurs.
Nous avions alors :
Veilleur A : ObjetsA
Veilleur B : ObjetsB
Veilleur C : ObjetsC

Lorsque le veilleur A mourrait, un bug se produisait il était "remplacé par les joueurs ayant un ID supérieur.
Le veilleur B utilisait les ObjetsA, le veilleur C utilisait les objets B et les objets C n'était pas utilisé.

Équilibrer les défense permettait alors d'éviter le cas suivant :
ObjetsA : 20 de def
ObjetsB : 30 de def
ObjetsC : 120 de def.
Perte des 120 de def....

Actuellement, le bug est corrigé. Donc plus de soucis de ce coté.

Il peut être préférable de mettre le maximum sur un veilleur qui ne peut pas mourir ou minimiser les chances. Gardien / alcool / goule.
