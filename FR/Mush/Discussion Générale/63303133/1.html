Je sais pas trop. Ca dépendra de qui reprendra Twinoid au final. Voilà ce qui a été annoncé : Dead Cells repris par le nouveau studio (3 ou 4 personnes y bossent, de mémoire) et MT bosse sur de nouveaux projets (ils sont une 10e). A partir de là y'a plusieurs conjectures possibles (et c'est de la pure conjecture, j'ai aucune info).

1. Twinoid (et tous les jeux associés) sont aussi repris par le nouveau studio. Dans la mesure où le studio est petit, il a fort à parier qu'ils fassent que de la maintenance. On sort Dead Cells de l'équation parce qu'on sait déjà ce qu'ils font dessus (la même chose que MT avant sur les mises à jours gratuites) et on se focalise sur Twinoid.
Parce qu'ils sont petits, y'a fort à parier que tous les jeux qui rapportent rien dégagent : c'est du temps perdu pour 0 gain (c'est une société, ça doit se faire de l'argent...). Exit donc tous les jeux sauf DinoRPG, Hordes et Mush (dans le meilleur des cas). Maintenant "et ensuite" ?

Soit ils se focalisent à faire des mises à jour sur les jeux existants, soit ils reprennent les concepts pour en faire une version jouable sur Steam (c'est là où ils se font de l'argent; DC le prouve).
J'ai beaucoup de mal à croire que la première solution soit réaliste. Les codes sont anciens, ils dépendent de librairies qui sont vielles. C'est une des raisons qui fait que mettre à jour Twinoid (pour le bug de 2110 par exemple) a pris autant de temps : la correction est simple, mais littéralement retourner dans le passé en terme de librairies pour recompiler, c'est la misère. Donc quoi ? Mettre à jour le code pour être compatible avec les librairies récentes ? Pourquoi pas... mais ça me semble irréaliste. Les développeurs d'origines des jeux ne sont plus là. Reprendre le code d'autrui est un calvaire. Surtout si c'est du vieux code qui n'est plus en phase avec les standards actuels. Bref j'y crois moyen.

2. C'est MT qui garde Twinoid. Et là j'ai du mal à voir ce qui pourrait changer. Ils ont laissé tomber Twinoid pour Dead Cells et actuellement ils bossent sur des prototypes pour un future jeu Steam. Je vois pas pourquoi ils reviendraient sur Twinoid après Dead Cells plutôt que de se focaliser sur ce qui marche. Éventuellement, ils pourraient reprendre un concept de jeu Twinoid pour un nouveau jeu, mais ça ils pouvaient le faire avant aussi. Dans ce cas là, je vois surtout une fermeture complète de tous les services avec éventuellement une MAJ "finale" pour certains jeux où les dev' sont encore dans le coin (Hordes & DinoRPG)


Bref... On verra ce que Buzzard dit quand y'aura une annonce. Malgré le joli pavé que j'viens d'écrire, j'vois pas trop l'intérêt de faire des conjectures.
