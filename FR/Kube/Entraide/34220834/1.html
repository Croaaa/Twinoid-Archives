Ça ne l'est pas (où alors j'ai vraiment raté un truc, mais l'API est assez simple que que ce soit peu probable)

C'est facile de voir ce qu'il y a :
Il suffit d'aller sur l'appli de démo de l'API
Et de taper ça

me?fields=sites.filter(18).fields(stats.fields(name, score), achievements.fields(name, score, date))

Ce qui montre en gros toutes les stats disponibles pour Kube.

Il y a 2 sortes d'infos personnelles sur Kube : les "stats" et les "achievements".
Les "stats" donnent des nombres, des données quantitatives (nombre de coffres ouverts par exemples) ; c'est là qu'on aurait pu avoir "nombre d'EMV", mais non.

Les "achievements" donnent des informations qualitatives, binaires (on l'a ou on ne l'a pas). Dont le titre "buveur". L'info quantitative associée n'est pas le nombre d'EMV qu'on a bue, mais le nombre d'EMV qu'on a du boire pour avoir le titre (100 donc ; mais ça sera 100 même si on en a bu 10.000)

C'est évidemment (et c'est pas un hasard) très proche de ce qu'on voit déjà dans les stats twinoid.

Les choses qu'on peut voir via l'API alors que ne les a pas autrement, c'est soit pour les jeux qui ont une API spécifique autre que celle de twinoid (pour l'instant la liste est courte : Hordes)
Soit pour twinoid lui-même (seul "jeu" de twinoid pour qui on ne peut pas voir les stats dans la colonne de droite ; du coup l'API permet de connaître ses médailles qui ne sont disponibles nulle part ailleurs)
