Bonjour tout le monde,

Petite suggestion qui j’espère vous plaira (donc kuber un max le sujet afin de capter l'attention des Admin !).

Ce que je suggère, c'est la mise en place d'un API pour Monster Hotel.
Voici les référence :
http://monster-hotel.net/graph/me => types "User".
http://monster-hotel.net/graph/user/$id => types "User".
http://monster-hotel.net/graph/users/$id,$id,$id => types "Array<User>" Restriction => "Max : 50".
Voici les Types :
User {
 id: Int, //Id du joueur ... donc de l'hôtel
 name: String, //Nom du joueur
 clients: Int, //Nombre de client héberger
 Visites: Int, //Nombre de visite faite
 good_clients: Int, //Nombre de clients satisfait
 good_vip: Int, //Nombre de vip satisfait
 kill_clients: Int, //Nombre de clients tuer
 clients_voles: Int, //Nombre de clients volé
 tresor_trouve: Int, //Nombre de tresor trouver
 amour: Int, //Amour donnee
 soda: Int, //Soda donnee
 lessive: Int, //Lessive donnee
 savon: Int, //Savon donnee
 papier: Int, //Papier donnee
 salle_boost: Int, //Salle booster
}
Exemple :
User {
 id: 9079946,
 name: Alexis,
 clients: 922,
 Visites: 81,
 good_clients: 89,
 good_vip: 15,
 kill_clients: 104,
 clients_voles: 355,
 tresor_trouve: 0,
 amour: 524,
 soda: 1048,
 lessive: 491,
 savon: 216,
 papier: 731,
 salle_boost: 108,
}
Aucune donné ne nécessite de limitation puisque chacune des information et disponible publiquement.
Notes, le but de l'API est de permettre a des développeur comme moi de "crée" des application pour vos jeux préféré !

La possibilité d'utiliser une API sur Monster Hotel pour récupéré les donné des joueur afin de "créer des Applications (sous la forme de sites Web) qui complètent ou améliorent l'expérience des joueurs" :3
Notes, pour les p'tit malin qui vont me dire que l'on peut déjà récup toute ces donnée via les récompense twinoid ... JE DIT NON !
Pour deux raison :
- Je trouve que c'est totalement stupide d'avoir a passer par un API tierce (celle de twinoid) pour récupéré des donné sur un jeux précis (Monster Hotel).
De plus, l'utilisateur "signe" pour donner ces donnée twinoid lors de l'utilisation de l'api twinoid, je ne trouve donc pas normal que l'application puisse accédé au donné 'autre' (monster hotel/dinoRPG/etc).
Le compromis idéal serais que l'application ne puise recevoir les donné des récompense d'un jeu de l'utilisateur que si l'utilisateur autorise la transmission de ces donné (c'est a dire qu'il faut rajouter dans scope tout les jeux sur lesquels on veut recup's des donné [même si ce ne sont que des donné de récompense]).
Et enfin second point
- Les valeur des stats sur le jeux (la sorte d'étoile) ne sont pas les même que les stats de récompense.
