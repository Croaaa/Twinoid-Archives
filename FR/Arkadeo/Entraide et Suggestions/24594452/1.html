Hmmm. La démonstration mathématique n'est pas bien compliquée, ou alors c'est le soir ? ^^

En partant du fait qu'une ligne de 4 donne 400 points (4 x 100), une ligne de 5 donne 1000 points (5 x 200), 5 donne 1800 points ( 6 x 300), etc... vient la formule :

Total = n x (n - 3) x 100, où n est le nombre d'animoz en contact.





Reste le problème des "croisements". On peut voir un premier combo de 7 par 8, qui me donne des animoz à 1200 chacun, pour un total de 16800. Selon ma formule, je devrais avoir un n de 14 (l'animoz du milieu n'est pas compté deux fois). Je devrais avoir 15400 points au total, ma formule est donc fausse pour les croisement. Si je compte l'animoz du milieu deux fois, n = 15, je me retrouve avec 18000, encore faux. La bonne formule est (14 x 12 x 100), c'est à dire n = 14, un nouveau "n", rebaptisé a + b, doit être égal à 15. Si on ajoute la "taille" de chaque ligne, on arrive à ce résultat.

On adapte donc la formule :

Total = n x ( [ a + b ] - 3 ) x 100 où :
n est le nombre d'animoz en contact,
a est la taille de la ligne 1,
b est la taille de la ligne 2.
Tout animoz au croisement d'une ligne est compté dans la taille de la ligne, mais n'est compté qu'une fois pour n.





Cette formule fonctionne pour tous les croisement de 2. Mais pour les croisement de 3 ou plus, impossible de trouver une logique cohérente. Les animoz commencent à avoir des valeurs différentes, 900 ou 700 ou 1000, dafuq !

La seule esquisse possible serait non plus mathématique mais du point de vue du code :
S'il y a plus de 2 lignes d'animoz, prendre la plus grande ligne, puis la plus grande la coupant. Calculer les points suivant la formule générique (avec n).
Les autres lignes sont des "bonus" additifs, pas multiplicatifs (ils ne modifient pas n). On compte la valeur de chaque animoz restant suivant la formule :
V = ( [ a + b ] - 3 ) x 100 où :
a la taille de la ligne 1,
b la taille de la ligne 2.





On arrive donc à la formule finale :
Total =
n x ( [ a + b ] - 3 ) x 100
+ n1 x ( [ a1 + b1 ] - 3 ) x 100
+ n2 x ( [ a2 + b2 ] - 3 ) x 100
+ n3 x ( [ a3 + b3 ] - 3 ) x 100, etc...
où :
n est le nombre d'animoz en contact dans la plus grande ligne et sa plus grande sécante,
a est la taille de la plus grande ligne (désignée L),
b est la taille de la plus grande ligne coupant L.

n1 est le nombre d'animoz en contact dans une hypothétique troisième ligne, n'ayant pas encore "reçu" de score.
a1 est la taille de la ligne verticale concernée (arbitraire),
b1 est la taille de la seconde ligne concernée (arbitraire).

n2 est le nombre d'animoz en contact dans une hypothétique quatrième ligne, n'ayant pas encore "reçu" de score.
a2 est la taille de la ligne verticale concernée (arbitraire),
b2 est la taille de la seconde ligne concernée (arbitraire).

n3 est le nombre d'animoz en contact dans une hypothétique cinquième ligne, n'ayant pas encore "reçu" de score.
a3 est la taille de la ligne verticale concernée (arbitraire),
b3 est la taille de la seconde ligne concernée (arbitraire).





Exemple avec mon problème :
http://img833.imageshack.us/img833/43...
La plus grande ligne (L) en rouge, la plus grande ligne coupant L en vert, ligne 3 (n1) en orange, ligne 4 (n2) en violet. Leur taille est affichée.

On a deux lignes "supplémentaires", on utilise donc la formule avec n, n1 et n2.

Total =
n x ( [ a + b ] - 3 ) x 100
+ n1 x ( [ a1 + b1 ] - 3 ) x 100
+ n2 x ( [ a2 + b2 ] - 3 ) x 100

On remplace... On n'oublie pas qu'un animoz au croisement de ligne ne compte qu'une fois pour n.

Total =
12 x ( [ 8 + 5 ] - 3 ) x 100
+ 3 x ( [ 8 + 4 ] - 3 ) x 100
+ 4 x ( [ 5 + 5 ] - 3 ) x 100

=

12 x 1000
+ 3 x 900
+ 4 x 700

=

17500, confirmé par le replay.
Pour les flemmards, le screen : http://img703.imageshack.us/img703/35...


Conclusion : au-delà de deux lignes se croisant, les lignes supplémentaires ne sont pas vraiment rentables. Il vaut mieux faire 2 grandes lignes pour maximiser son multiplicateur de score n, plutôt que d'ajouter des points avec des lignes supplémentaires qui ne sont pas multiplicatives.

Houlà, ton explication est vraiment trop longue et trop compliquée... Le détail des scores est prévu dans l'aide.

n : nombre d'animoz dans la ligne
Score d'un animoz dans une ligne simple (n - 3)*100
Score d'un animoz dans une ligne double (n - 2)*100


Lorsque le nombre de lignes est supérieur à deux. Le calcul se fait sur chaque ligne séparée et considère que ce sont plusieurs lignes doubles.
Pour chaque ligne, la ligne la plus longue qui la croise est prise en compte et le score est celui d'une ligne double.

Donc oui, deux longues lignes valent plus que trois lignes ou une ligne avec le même nombre d'animoz.
