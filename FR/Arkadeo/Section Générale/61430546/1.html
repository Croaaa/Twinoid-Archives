Informations diverses sur le problème

Le problème dont il est question dans ce sujet sont les lags de Mel dès la première seconde de jeu, rendant le jeu complètement injouable.
C'est lié à la version 30 de flash qui inclus un ralentissement d'une fonction interne pour se protéger des failles Meltdown/Spectre. Plus de détails ici.

Pour faire court, la solution consiste à modifier (ou créer s'il n'existe pas) le fichier de configuration flash "mms.cfg" dans le répertoire qui va bien et d'y ajouter la ligne "TimerJitterMicroseconds=0". Cette action désactive la mise à jour de sécurité rétablissant les performances de la fonction interne.

Ce fichier peut se trouver dans les répertoires suivant sur Windows :
C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Pepper Data\Shockwave Flash\System (répertoire System à créer, répertoire spécifique à Chrome)
C:\Windows\SysWOW64\Macromed\Flash (OS 64bits)
C:\Windows\System32\Macromed\Flash (OS 32 et 64bits)
Sur Mac il se trouverait ici, non testé :
/Library/Application Support/Macromedia/mms.cfg
Sur Linux, cherchez sur google, cela doit dépendre. Il peut être ici :
/etc/adobe/mms.cfg

Après la modification du fichier, une simple relance du navigateur suffit pour prise en compte, et c'est vérifiable en regardant un replay Mel.


Mise à jour automatique du fichier par script pour Windows :

Télécharger le zip à ce lien : gist.
Deux scripts présent :
- Flash_DisableJitter_Chrome_Windows.bat : pour chrome 64bits, à lancer simplement en double cliquant
- Flash_DisableJitter_OtherBrowser_Windows.bat : pour tous les autres cas, à lancer en faisant clic droit / exécuter en mode Administrator

Mise à jour manuelle pour Windows pas à pas :

[[ Google Chrome ]]
vérification facultative :
Copier dans la barre d'adresse ce lien : chrome://version/
Vérifier la ligne flash qu'elle pointe bien un fichier dans AppData/Local, en l'occurrence pour moi : "Flash: 30.0.0.149 C:\Users\USER\AppData\Local\Google\Chrome\User Data\PepperFlash\30.0.0.149\pepflashplayer.dll".

Opérations à effectuer :
- Aller dans le répertoire suivant en remplaçant USERNAME par votre login de session : "C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Pepper Data\Shockwave Flash"
- Créer le dossier "System" à l'intérieur s'il n'existe pas déjà
- Aller dans le dossier System, y créer le fichier "mms.cfg" s'il n'existe pas déjà. Pour le créer, faire clic droit / nouveau document texte, donner lui un nom bidon.
- Ouvrir le fichier avec le bloc note, copier la ligne "TimerJitterMicroseconds=0" (sans les guillemets) dedans, puis faire Fichier / enregistrer sous / mettre en nom de fichier "mms.cfg"
- Vous pouvez ensuite supprimer le fichier txt bidon.
- C'est terminé, relancé chrome et vérifier que sur un replay Mel fonctionne bien.

[[ Autres navigateurs ]]

Vérifier la présence d'un fichier nommé "mms.cfg" dans les répertoires C:\WINDOWS\system32\Macromed\Flash\ et C:\WINDOWS\sysWOW64\Macromed\Flash\
Sur un OS 32bits, vous ne le trouverez logiquement que dans le premier dossier. Si vous ne trouvez aucun des répertoires, y a un soucis, inutile d'aller plus loin

Opérations à effectuer :
- Si un fichier mms.cfg est présent, copier le ailleurs (par exemple dans mes documents ou sur le bureau).
-> Ouvrir avec le bloc note le nouveau fichier dupliqué
-> Ajouter une nouvelle ligne contenant "TimerJitterMicroseconds=0" (sans les guillemets), sauvegarder et fermer le bloc note.
- Sinon, si aucun fichier mms.cfg est présent, créer un nouveau fichier sur le bureau comme expliqué dans le paragraphe pour google chrome.
- Copier/Coller le fichier mms.cfg modifié vers les répertoires C:\WINDOWS\system32\Macromed\Flash et/ou C:\WINDOWS\sysWOW64\Macromed\Flash (valider la pop-up demandant de confirmer les droits d'admin).
- Vous pouvez supprimer le fichier dupliqué (sur le bureau ou mes documents).
- Relancer le navigateur et vérifier que sur un replay, Mel fonctionne bien.

[[ Problèmes remontés ]]

Il a été remonté sur que un OS 32bits, cela ne fonctionne pas sur chrome avec les 2 fichiers modifiés (dans le AppData/Local et dans System32). Aucune explication n'a été trouvée, mais cela fonctionne pour Firefox.

Du coup pour résumer, pour pallier aux lenteurs de mel, il faut ajouter la ligne suivante dans le fichier "mms.cfg" :
TimerJitterMicroseconds=0

De mon côté qui est sur Windows 10 64bits.

Chemin du fichier pour Edge, IE, Firefox -> C:\Windows\SysWOW64\Macromed\Flash
Le dossier étant protégé, le plus simple est de copier le fichier ailleurs, ajouter la ligne, puis refaire la copie inverse (peut-être que sur un système ou navigateur 32 bits, il faut modifier le fichier du dossier "System" au lieu de SysWOW64).

Chemin du fichier pour Chrome en remplaçant username par son login -> C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Pepper Data\Shockwave Flash\System\mms.cfg
Pas de droit spécifique pour éditer ce fichier. Cependant il y a des chances qu'il faille le créer plutôt que l'éditer (avec le dossier System qui va avec).

Une relance du navigateur est censée suffire pour prise en compte (voir même une relance de l'onglet).

Remarque : cette action désactive une mise à jour censé protéger contre les failles Meltdown/Spectre. Si vous n'avez pas confiance aux applis flash de la MT, ne le faites pas.

Informations diverses sur le problème

Le problème dont il est question dans ce sujet sont les lags de Mel dès la première seconde de jeu, rendant le jeu complètement injouable.
C'est lié à la version 30 de flash qui inclus un ralentissement d'une fonction interne pour se protéger des failles Meltdown/Spectre. Plus de détails ici.

Pour faire cours, la solution consiste à modifier (ou créer s'il n'existe pas) le fichier de configuration flash "mms.cfg" dans le répertoire qui va bien et d'y ajouter la ligne "TimerJitterMicroseconds=0". Cette action désactive la mise à jour de sécurité rétablissant les performances de la fonction interne.

Ce fichier peut se trouver dans les répertoires suivant sur Windows :
C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Pepper Data\Shockwave Flash\System (répertoire System à créer, répertoire spécifique à Chrome)
C:\Windows\SysWOW64\Macromed\Flash (OS 64bits)
C:\Windows\System32\Macromed\Flash (OS 32 et 64bits)
Sur Mac il se trouverait ici, non testé :
/Library/Application Support/Macromedia/mms.cfg
Sur Linux, cherchez sur google, cela doit dépendre. Il peut être ici :
/etc/adobe/mms.cfg

Après la modification du fichier, une simple relance du navigateur suffit pour prise en compte, et c'est vérifiable en regardant un replay Mel.

Télécharger le zip à ce lien : gist.
Deux scripts présent :
- Flash_DisableJitter_Chrome_Windows.bat : pour chrome 64bits, à lancer simplement en double cliquant
- Flash_DisableJitter_OtherBrowser_Windows.bat : pour tous les autres cas, à lancer en faisant clic droit / exécuter en mode Administrator

Mise à jour manuelle pour Windows pas à pas :

[[ Google Chrome ]]
vérification facultative :
Copier dans la barre d'adresse ce lien : chrome://version/
Vérifier la ligne flash qu'elle pointe bien un fichier dans AppData/Local, en l'occurrence pour moi : "Flash: 30.0.0.149 C:\Users\USER\AppData\Local\Google\Chrome\User Data\PepperFlash\30.0.0.149\pepflashplayer.dll".

Opérations à effectuer :
- Aller dans le répertoire suivant en remplaçant USERNAME par votre login de session : "C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Pepper Data\Shockwave Flash"
- Créer le dossier "System" à l'intérieur s'il n'existe pas déjà
- Aller dans le dossier System, y créer le fichier "mms.cfg" s'il n'existe pas déjà. Pour le créer, faire clic droit / nouveau document texte, donner lui un nom bidon.
- Ouvrir le fichier avec le bloc note, copier la ligne "TimerJitterMicroseconds=0" (sans les guillemets) dedans, puis faire Fichier / enregistrer sous / mettre en nom de fichier "mms.cfg"
- Vous pouvez ensuite supprimer le fichier txt bidon.
- C'est terminé, relancé chrome et vérifier que sur un replay Mel fonctionne bien.

[[ Autres navigateurs ]]

Vérifier la présence d'un fichier nommé "mms.cfg" dans les répertoires C:\WINDOWS\system32\Macromed\Flash\ et C:\WINDOWS\sysWOW64\Macromed\Flash\
Sur un OS 32bits, vous ne le trouverez logiquement que dans le premier dossier. Si vous ne trouvez aucun des répertoires, y a un soucis, inutile d'aller plus loin

Opérations à effectuer :
- Si un fichier mms.cfg est présent, copier le ailleurs (par exemple dans mes documents ou sur le bureau).
-> Ouvrir avec le bloc note le nouveau fichier dupliqué
-> Ajouter une nouvelle ligne contenant "TimerJitterMicroseconds=0" (sans les guillemets), sauvegarder et fermer le bloc note.
- Sinon, si aucun fichier mms.cfg est présent, créer un nouveau fichier sur le bureau comme expliqué dans le paragraphe pour google chrome.
- Copier/Coller le fichier mms.cfg modifié vers les répertoires C:\WINDOWS\system32\Macromed\Flash et/ou C:\WINDOWS\sysWOW64\Macromed\Flash (valider la pop-up demandant de confirmer les droits d'admin).
- Vous pouvez supprimer le fichier dupliqué (sur le bureau ou mes documents).
- Relancer le navigateur et vérifier que sur un replay, Mel fonctionne bien.

5) Dans ce dossier System, créer un fichier mms.cfg avec comme contenu "TimerJitterMicroseconds=0". En ligne de commande sur linux pour créer le fichier avec cette ligne : echo "TimerJitterMicroseconds=0" > mms.cfg
