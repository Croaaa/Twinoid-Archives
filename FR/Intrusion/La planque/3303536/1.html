Je m'étais toujours dit que les noms des terminaux figurant dans un fichier .reseau devaient suivre un ordre logique. Effectivement, après plusieurs tentatives, j'ai trouvé comment repérer un poste à partir de ce fichier.

Le premier de la liste est le poste situé en bas à droite. Le suivant est celui au dessus de lui (en suivant une ligne diagonale \ ), indépendamment des connexions.

Je m'étais aussi posé la question de savoir pourquoi certains firewall possédaient tous le réseau dans leur fichier .reseau et d'autre pas. Jusqu'à preuve du contraire, le fichier .reseau d'un terminal donné contient la liste de tous les terminaux situés après lui. Dans l'exemple ci-dessous, le fichier .reseau du poste G ne contiendrait que H, tandis que celui de C serait vide.

Le schéma est à tourner de 45° dans le sens contraire des aiguilles d'une montre. Les ¨¨ sont juste des espaces.

¨¨¨¨¨¨¨¨¨¨F-----+-----C
¨¨¨¨¨¨¨¨¨¨¨¨¨¨ |
H-----G-----E-----+------B------A
¨¨¨¨¨¨¨¨¨¨|¨¨¨¨|
¨¨¨¨¨¨¨¨¨¨D¨¨¨
¨¨¨¨¨¨¨¨¨¨¨¨¨¨¨|
¨¨¨¨¨@------------+-----X-----X

A -> F: terminaux
: Firewall dont est tiré le fichier .reseau
@ : point de départ
X : terminaux ne figurant pas sur le fichier .reseau

Voilà, j'espère que cette analyse pourra vous être utile. Bon piratage!
