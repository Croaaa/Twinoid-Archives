Démonstration!

#HACKRC de Qwerty800

alias ./$file = target $file
#Notez que je viens dé créer la variable file!
#tout ce qui se trouve après le "./" sera consitédé comme le nom de mon fichier!

alias rm -f $file = dmgm $file
#Idem ici! J'utilise un espace, ce qui serait normalement considéré comme faisant partie du nom du fichier!

#Attention, on y va pour une macro!
alias av $file = dmgm $file; dmgm $file; dmgs $file
#Cette macro permet de, en une seule commande, détruire tout anti-virus dans mon chemin!
#Notez l'usage du point-virgule, qui permet normalement d'enchainer plusieurs commandes à la fois.

alias mis = cp $target && rm $target
#Voyez comment j'utilise la double esperluette, encore une fois pour enchaîner des commandes!
#J'utilise aussi la variable target, pour copier et effacer l'objet de ma mission.

Je vais répondre ce que j'ai répondu sur le support à une question similaire :

Nous avons choisi de limiter les possibilités de la ligne de commande (pas de Pipe par exemple) pour ne pas lui accorder une puissance vraiment trop importante.

Pour information, historiquement, il était possible de rédiger de petits scripts à une époque dans le mode ligne de commande (conditions, boucles, exécutions...). Le résultat était certes intéressant d'un point de vue "réalisme", mais démesurément trop facile à jouer en chainant les bons scripts.

Pour même aller encore plus loin, il faut savoir qu'Intrusion (version 0.1) était un jeu en ligne de commande pure, avec un éditeur de programmes (macros). Après quelques tests, le jeu s'est révélé pas super intéressant à jouer, sauf pour les puristes D'où le basculement en "mode graphique".
