Bon, comme cela a été dit dans de nombreux sujets, le mode Uber Eleet, c'est bien, ça permet de gagner du temps, et ça roxe. Youpi.

Mais je trouve qu'il faudrait améliorer la commande target.
On a très souvent besoin de cibler tous les fichiers d'un dossier, un par un. (recherche de numéros de CB, recherche de mails pour une mission, ...)
Pour effectuer ce parcours, la commande target est super pas pratique. Même avec des alias, et même avec la complétion automatique. On se retrouve à devoir taper des chiffres, le TAB, re-des chiffres, re-TAB. etc.

Du coup, on utilise la souris pour cibler les fichiers, même en mode Eleet, et c'est quand même dommage. Si on pouvait s'en sortir QUE avec des lignes de commandes, ça serait vachement mieux.

Donc je propose l'ajout de paramètre pour la commande target.
target -next : cible le fichier situé après le fichier courant. (Si aucun fichier n'est ciblé, la commande ne fait rien. Si le fichier courant est le dernier, la commande ne fait rien non plus)
target -prev : cible le fichier précédent.

Et un truc encore plus mieux :
target -next 5 : cible le 5ème fichier situé après le fichier courant. (ça permettrait de faire défiler l'écran plus vite, pour les dossiers contenant beaucoup de fichiers.)
target -prev 5 : cible le 5ème fichier situé avant le fichier courant.

Et un truc encore mieux mieux :
target -next -filter *mail : cible le fichier suivant dont le nom correspond à *mail (c'est à dire les fichiers se terminant par mail).
target -next 3 -filter *mail : cible le troisième fichier à partir du fichier courant, dont le nom se termine par mail.

Avec une commande target de ce genre, et quelques alias bien choisi, on pourrait tout faire péter !!!
