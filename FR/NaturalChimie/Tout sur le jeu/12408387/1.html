Bonjour,

C'est vrai qu'on peut manipuler la fenêtre d'affichage en javascript, même si ce n'est pas un accès aux onglets de manière globale, ça permet de mettre une confirmation sur la fermeture de l'onglet courant. Ce qui est finalement tout ce que vous demandez

Néanmoins manipuler la fenêtre d'affichage, ce n'est pas très très "propre", donc je persiste à penser qu'une extension pour, par exemple, verrouiller un onglet - à l'initiative de l'utilisateur - est la meilleure façon de faire pour les gens qui le souhaitent

Je suis d'accord sur le fait que ça pourrait être utile à des gens qui n'iront jamais chercher une extension pour leur navigateur, mais :
- c'est typiquement le genre de chose dont les accès en javascript dépendent énormément du navigateur, d'où la notion de "pas très propre".
- D'un point de vue déontologie du web, je pense qu'il est préférable que les sites web se cantonnent à l'intérieur de leur page web et ne fassent rien au-delà, et à l'inverse que les navigateurs affichent les pages demandées mais ne les trafiquent pas.
Un exemple inverse de ce type : les extensions anti-pub. C'est très pratique et ça marche plutôt bien, mais leur principe est complètement destructeur pour pas mal de sites. Ca ne doit pas être plus agréable pour les développeurs des navigateurs qu'un site "bidouille" le comportement de leur logiciel

Cordialement,
L'équipe Naturalchimie
