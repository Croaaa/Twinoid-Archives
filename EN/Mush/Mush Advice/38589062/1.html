DraconicEpicWhat I don't understand is why they don't roll back to a previous version of the script till the problem is solved. Any good software project, web or other wise, tends to have some form of version control for just this type of situation.

Because that page is interconnected with the game itself. To roll it back safely they'd have to roll back the game, roll back any changes to the database schema, and potentially roll back the data itself (if any data valid in the new version would have been invalid in the old one). Rolling back just one page of a tightly-interconnected system is incredibly error-prone even with version control because a change in the schema or core files will usually impact multiple pages. Do you wanna risk losing your current ship? Do ya?

Besides, don't they already have a fix on the other servers? If they were going to take the time to do something for this server, why wouldn't they just apply the fix?
