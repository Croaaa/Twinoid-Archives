On august 9, 2011
Kompera
Turning off the firewall is only to verify if the problem is firewall-related.
While this is a reasonably sound troubleshooting step on the face of it, it has a couple of critical flaws:

1) Unless something has changed within the firewall settings (changes end users do not typically make), the change from successful connectivity to failed connectivity should not be blamed on a component which has not changed. Look elsewhere first.
2) Advising an end user to disable their firewall invites disaster. Many end users run nothing by way of security aps except the OS provided firewall which is on by default (for XP SP 2 and later MS OS releases). Without the firewall, they are very vulnerable. The average time to infection of a Windows XP PC connected to the internet w/o a firewall is 4 minutes. This has hopefully gone way up with Windows 7, but the point remains the same: any PC w/o a firewall is subject to nearly immediate attack.
