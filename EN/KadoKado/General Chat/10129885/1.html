Being able to cancel an attack at any time is useful for people who are either in a small clan, or can only be online during specific timeframes

The problem with any "locking" on attacks if someone is defending it is that there's currently no support for something like that. I am only assuming that the website doesn't keep track of "active defends", though

Anyway, that "locking" can be abused. If you have it where it sets a lock when a game starts, someone could just start a game, and close that tab (not loading any KK pages after that). If it's something where the game SWF pings the server every few minutes, someone could just run something to slow their game down to a crawl

The point of keeping a lock on for a long time is to allow time for someone else to repel a "bad" score. If a lock also prevents someone else from defending, then it would just be a way to grief the attacker. Griefing like that will slowly cause attackers to stop targeting a particular clan, since it's annoying to be required to get a great score on the first try

There can't be a time limit for the lock, either. There are a few untimed games on here

I say that if you want the lock system, there should be a 30 minute grace period, where an attack can be cancelled even if a lock is active. That way, any reasonable person won't be affected negatively
