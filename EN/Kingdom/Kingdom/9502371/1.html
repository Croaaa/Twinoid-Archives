I like to report a glitch in the game that I already discovered two times, which allows traveling of a general where it should not be possible. Let´s take an example case, with P1 = Player 1, P2 = Player 2, no passage rights, and both have a wall so other players cannot pass the others territory.
-
[P1 Capital] - [P1 territory] - [X1 free territory] - [P2 territory] - [P2 capital]
-
What is correct is that if I try to send a general from [P1 Capital] to [P2 capital], it does not work, because I can only reach [P2 territory] from outside. Also, if [X1 empty] has some barbarian troops, the general will automatically stop there although the initial route has been selected.
-
The error is now here: If I start a general from [P1 capital] and send it to [P2 territory], and while the general is on the way (often 12 hours), Player 2 conquers [X1 free territory], then the general should not be able to pass [X1 now P2´s new territory] because it now belongs to Player 2 and should not leave other players through. But instead of stopping, the general is just hopping over the [X1 now P2´s new territory], and marching directly to [P2 territory] without stopping. So it´s simply walking through an enemy territory and skipping one battle, although this should not be possible.
-
In the opposite case, if a route has been calculated for territories with passage rights and the general is on the way, and a territory is lost by someone because the kingdom is destroyed and now there are barbarian troops, the general stops. But in the upper case if a player takes a territory and put troops there, it is ignored and the general keeps traveling through enemy territory. It seems that the source code that handles the generals movement performs a check if barbarian troops are in the way at every hop, and then stops movement, but it does not check if the hop now belongs to a different player, and if there is a wall and/or troops to stop the general as well. It seems that the source code here only relies on the initially computed route, and with that being okay only checks for the barbarian troops to stop the traveling general.
