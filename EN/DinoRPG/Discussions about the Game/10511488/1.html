If MT officially states that they are opposed to scripting, than it's an easy decision for us, but we're still waiting on the official word.

I think some of you think that the Merguez script is unfair because it wasn't published publicly. By that, I don't meant just posting a thread in the forum, but making it a sticky so it doesn't get buried, and never to be retrieved again by the limits of the search function. The point that if scripting gives an unfair advantage over other players is cheating, is of course a valid argument, but that's for evaluating each script individually, not a blanket application to all programming and label them all as cheats.

Let's think of all scripts/game play methods that enhance our gaming experience, without giving unfair advantages to anyone, as User Created Contents. I'm sure most of you who've played any current gaming titles in the past few years know what that term is.

Basically, User Created Contents are things creating by the players community to enhance the gaming experience, and/or to extend the gaming life of a title.

In the Merguez scripting case, it simply is a small function for DinoRPG, but I see it as part of the user created content, as it improves gameplay, withing giving you undue advantage over other players.

Most of you have used User Created Content in this game already. Think of the skill tables, the missions/quests walkthrough, the location and dungeon maps, etc. They all shorten your time spent on finding things in this game, it's just that they are html scripts and graphics encoding.

Of course, MT will have to embrace this concept in order for it to work. MT will have to evaluate each script to verify the functions, approves it, then publishs it for the community. At that point it is up to each of us to decide on whether to use the script or not.
