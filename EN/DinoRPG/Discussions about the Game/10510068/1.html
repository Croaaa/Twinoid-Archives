Of course, translation could be really mucking this up (and I think it is), but here's the way I understood it to be calculated, starting without any additional skills:

10+10/1=20

10 being initial time value
10 being a constant
1 being speed
20 being the new time value

Round 1: time value = 10
Round 2: time value = 10+10/1 = 20
Round 3: time value = 20+10/1 = 30

Here's with Celerity (+15%):

10+10/(1+.15)=18.7

Round 1: time value = 10
Round 2: time value = 10 + 10/(1+.15) = 18.7
Round 3: time value = 18.7 + 10/(1+.15)= 27.4

Here's with Giant (-20%):

10+10/(1-.2)=22.5

Round 1: time value = 10
Round 2: time value = 10 + 10/(1-.2) = 22.5
Round 3: time value = 18.7 + 10/(1-.2)= 35

The Dinoz with the lower time value attacks that round, then adds to its time value. The game checks this new time value against the other Dinoz and whoever is lower will again be the one to attack that round.

So two Dinoz, one with no skills (A), the other with Celerity (B):

A:B
10:10 (one randomly goes first, lets say the no-skills Dinoz)
20:10 (Celerity Dinoz has the lower time value and thus goes this round)
20:18.7 (Celerity Dinoz is still lower and thus goes again this round)
20:27.4 (no-skills is finally lower and goes this round)
30:27.4 (Celerity is again lower and goes this round)

And so on....

If speed is like all other attributes, as I assume it would be, then speed upgrades dont just add to each other.

In the case of lightning attack and adrenaline:

Speed = 1.5 * 1.4 = 2.1

And if you add in celerity:

Speed = 1.5 * 1.4 * 1.15 = 2.42

Now the tricky part would come in to play when you have negative speed. For example Tai-Chi decreases speed of air attacks by 20%:

Speed = 0.8 = 0.8

Adding in Awakening (decrease speed of air attacks by 20%) and lonely meditation (decrease speed of air attacks by 50%):

Speed = 0.8 * 0.8 * 0.5 = 0.32

Since the speed is a percentage increase you cant just add the percentages to achieve the final speed.

Lastly the effect of having both a +speed and a -speed, shown by Celerity (+15% speed) and Giant (-20% speed):

Speed = 1.15 * 0.8 = 0.92

So as you can see its not just the 5% difference that you end up losing.

Again, I'm not sure if this is how the programming actually works, it just makes logical sense to me this way.
