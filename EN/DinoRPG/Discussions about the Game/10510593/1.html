I believe the dmg calculation involves subtracting the enemies defence, so you cannot use ratios to compare the dmg done with different element values.
I think the dmg is calculated something like this:
(Assault Power - Defence) / Constant
Note: the "Constant" is to reduce the dmg done in pvp fights, I think it would be much smaller value when fighting monsters.
Take the case of burning breath (with fire element and tornado (with air element 12). Lets also assume an enemy defence of 20.
Burning Breath DMG: (5*8 - 20) / Constant = 20/C
Tornado DMG: (5*12 - 20) / Constant = 40/C
In this case, tornado does twice the dmg even though the element value is only 1.5 times greater.
If you change the value of the enemy defence you'll find that as defence increases the ratio between dmgs increases (if you keep the same element values). For example, increasing the defence in the above example to 30, would give burning breath a dmg of 10/C and tornado a dmg of 30/C (3 times greater). Likewise, if you decrease the defence, to 10 for example, burning breath dmg would be 30/C and tornado dmg would be 50/C (1.66 times greater).
So, as you can see, you cant just use ratios to determine how much more dmg a skill will do.

Skills like strength would be factored in to your assault power.
The problem with comparing skills like burning breath and tornado is that they are different elements so enemies will have different defences.
You need to compare skills of the same element against the same enemies to ensure the same assault and defence numbers are being used.
As for blazing twilight, it must take in to account both the fire and lightning defence in some weighted average.
The biggest problem, however, will be the fact that there is randomness to every assault in terms of the dmg done. If a dino performs an A skill twice in a fight, the dmg done almost always changes ... with this random aspect youll never be able to prove how dmg is really calculated.
