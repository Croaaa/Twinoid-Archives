calorifere
I have the experience in every game that I play in which randomness is involved, that people always misunderstand what randomness means, and that they always blame the random number generator, hackers, or modders for extended sequences of luck / badluck.

The probability that the random number generator, here or anywhere else, is badly programmed is exceedingly low. Random number generators are pretty well understood objects by now, programmers for websites like this one will usually use the "default" random number generator for whichever programming language they're using, and those random number generators are usually perfectly all right.

It's selective memory and the human brain's overwhelming desire to find patterns in everything that are playing tricks on you. This happens all the time.
In perfectly random events, there occasionally be long (sometimes even, VERY long) sequences in which people will seemingly be abnormally lucky or unlucky. This is in fact prefectly normal (it would in fact be abnormal if it didn't happen). Get used to it.

Husaurus
I guess I wasn't clear enough. If you have two calls to a random number generator that trigger a fix number of clock cycles apart, the second random number will not be independently 'random' from the first... in fact, it will result in a pattern, knowing the first time seed in such a series of calls, an entire series of random numbers can be predicted.

Repeated calls to a random number generator can form patterns when there is a pattern to the repeating. Granted, there would be tens of thousands of patterns, at least one for each initial random seed... but they are still there.

Does it matter? Not really, especially for a game.
