Here is a big block, which summarizes the situation of all Eternal-Twin.
I should point out that my current position is to help with the MyHordes project, but I help a little with all the other games currently online as i can.
Overall we are short of developers for a lot of games so everyone is welcome, but I'll let you read everything.
We are also thinking about a simpler method to let know the progress of a particular game without having to open various topics, but in the meantime I'm creating this topic dedicated to this.

Summary of the progress of the Eternal-Twin project as a whole as of September 26, 2020 :

About the EternalTwin website (according to Demurgos ) :

The Forum : The next update should complete the forum, which is already available but still lacking in features. We could get a forum like Twinoid's not long from now. ﻿
The account : It will soon be possible to edit your nickname or to change mail/passphrase, with a complete and functional account configuration page. ﻿
The website itself : Pictograms and blocks are coming soon. ﻿

In Summary: The deployment script should get stable and be published soon, which means the website would be usable by the games and users.

Concerning the games :

"Almost finished" games for which most of the work has been done :

StudioQuizz : Game almost completely reproduced, with only a few issues to fix and a reward system through the integration to EternalTwin. Still the game is playable. A playable version of the game is already available. Niko300 (Nikofly) is in charge of this version.
Majority : A playable version of the game is already available, without integration however to EternalTwin. It will be done eventually. At the moment, the game is not hosted on a server.

Games with a lot of progress done :

DinoParc : Jowpacabra (Jonathan) is progressing really smoothly, he already finished the fights, actions and cards, along with a lot of small things leading us to think the project will reach a good ending ! At the moment, the game is not hosted on a server.
Die2Nite : The project MyHordes is also progressing smoothly, its community is active and contribute well. A playable version is available, which is still getting enhanced, will be integrated to EternalTwin once completed, the possibility of importing its soul from Die2Nite as well as its variants is already possible. Brainbox is in charge of this version.

Games in progress :

CaféJeux : 20000Volts is progressing steadily, a lot of the website features being done. It's going step by step toward completion. At the moment, the game is not hosted on a server.
Kube : A game is not being developed, but the community is focusing on saving the current map and retrieving as many game elements as possible, in order to make sure the game can be recreated after it gets closed.
Fever : As with Kube, the community is mainly retrieving game info, as we believe the game might be playable without internet, with almost all, if not all, playable content available directly in the SWF (game's code), which has already been almost completely deobfuscated.
Mush : Different projects ongoing. Most of the efforts are put in retrieving game data, but the recreation of the game is also progressing.
DinoRPG : Game data collection has been going on, but there is no real progress currently. That being said, Jowpacabra might be able to help once he's done with DinoParc (Thanks to him!), which is reassuring regarding the project!
DinoCard : Not a lot of progress, but still a bit. Jowpacabra said he might (also) help if the project is stalling once DinoParc is done. Shivies is also helping. Most of the game info has been recovered at this time.
LaBrute : In progress by Scorpio and Nadawoo.


Stagnating games, with no progress, needing help or for which recreation has been postponed for various reasons :

MiniVille-AntiVille : Nadawoo is taking care of it, but the viability of the project is questioned. *﻿The possibility to integrate the game directly to EternalTwin to make towns from game progress is being considered.﻿*
Towerquest : No progress considering the lack of developer. We have the SWFs of the game but something is missing to be able to play the game.
Socratomancie : Nadawoo wishes to do it, but after other projects more important to him. No progress but a possibility for future development (which is better than nothing).
Intrusion : No progress yet.
NaturalChimie : Game data retrieval has been performed, Vicky_Z is in charge. That being said, it is a huge project and he would definitely need help, feel free to contribute if possible
KingDom : No developer so far (so no progress), but people are discussing the stack to use.
Snake : Alceste_ is in charge but it will take time. No progress yet.
Teacher-Story : No progress yet.
Hyperliner : No progress yet.
G55 : No progress yet, despite some G55 restoration projects having existed in the past.
Carapass : No progress yet.
Skywar : Gentilledoux is in charge but has not progressed yet.

KadoKado/Arkadéo : The games code might be retrieved at the end of the bidding period in September. The project members are currently focusing on the front and back ends of the website rather than on the games.

Games without progress, leader or developer :

No project leader or even dev. The recreation of these games seems unlikely. If people have the ability and wish to help one of them, they should come out:

MonsterHotel, RockfallerJourney, Streetwriter, CroqueMotel, Motion Ball 2, Odyssey,
MonLapin (Game data partially retrieved),
FruitiParc (Considering asking FruitiZone for help)

I enclose the latest email from Skool (Motion-Twin Administration) from today with some IMPORTANT news!

Hello,

I was able to add a few more elements to https://github.com/motion-twin/WebGam... :
- Burning Kiwi sources and what is normally FrutiSnake (the folder name was "snake3" but I think it fits)
- An SQL export of the official levels and tutorials of Pioupiouz
- A big part of the Hordes sources, which should help to find the rules of the game

I took the opportunity to include some files from Mush or DinoRPG that I had already sent by mail.

Concerning Twinoid and our online web games, we will unfortunately have to close them soon.
Their frequentation has become insignificant and keeping them online in the long term and in good conditions would require too much maintenance.

No date has been fixed for the moment, that's why nothing has been communicated until now.
It should happen within a few months I think, an announcement will be posted on Twinoid a few weeks before any closure.

It will be possible to put Eternaltwin in the spotlight at that time (and probably to put a link to the Eternaltwin site on the page that will replace our sites after their closure).

Finally, I would like to thank you for the energy you put into our old creations!
