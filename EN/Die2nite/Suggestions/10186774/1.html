I initially posted this message here in the Twinoid forum because I was not able to access the game. Of course, a few minutes later, the game becomes accessible again...



Here is an attempt at summarizing the problems that have affected die2nite since Friday and that have caused most towns to end prematurely.


On Friday at 23:00 (game time), the D2N attack starts as usual, but does not end.
About half an hour after the time at which the attack should have ended, players start reporting the problem here.
A few hours later, several other Twinoid become unavailable because of a database error. This affects all Muxxu games (migrated to Twinoid) in all languages. Bug reports are submitted to the technical support and the admins are paged in the middle of the night.
On Saturday, a bit after 11:00, the database server that was causing problems is rebooted and all games become available again, except for die2nite, which suffers from a different type of error.
During the afternoon, several players ask for die2nite to put on hold (quarantine) because most of the day is already gone and some players would not have time to reconnect before the attack even if the game is restarted. This is reported to the technical support and brought up to the admins.
At 20:48, die2nite is restarted and an announcement is posted: « Following an unforeseen technical problem, Die2Nite has been offline for most of the day We apologize for the inconvenience and we credited the cities some temporary defense points. » The towns get a bonus of 500 points, regardless of what they they are at.
Unfortunately, a new problem appears: most players who try to connect to the game get a blank page or a partially loaded page, which makes it impossible to play. This makes the problem worse than when the game was down, because this means that a new day has started and the attack will take place although the players are unable to camp, to close the doors of their town, to get rid of the dead bodies, etc.
After some investigation, several players discover a bit before the attack that the game server is sending corrupt files to most users: the JavaScript file "/js/horde2.js" that contains most of the game code is delivered as an empty file to all users of Firefox, while it seems to be delivered correctly to some other browsers. As a result, it is impossible for Firefox users to access the game (Firefox is the recommended browser for die2nite).
Some players succeed in accessing the game via the mobile version (Anthrax) but this can only be used to access the forums: it is not possible to camp, to move and return to town, nor to get rid of the dead bodies that would kill other players in town.
The attack takes place at 23:00, killing most of the towns because the players were unable to access the game.
After this "armaggeddon", several players publish some workarounds for accessing the game: use Firebug or a similar tool to force load horde2.js from another version of the game, edit the browser config settings to change the User-Agent so that the server thinks that one is using Chrome instead of Firefox, or switch to Chrome for playing the game. Unfortunately, this advice comes too late because most players are already dead.
On Sunday evening, the players who were not dead and who could still access the game discover a new problem: it is not possible to close the doors.


In summary, there were two main issues:

Some unknown error caused the game to be down during most of the day on Saturday.
The fix that was announced three hours before the attack on Saturday actually made the problem much worse: instead of putting the game on hold as requested by several players, the game was restarted but most players could not access it at all. This wiped out most towns, including those that were competing for the rankings.


Like many other players, I was affected by these problems and I died in a frustrating way because none of the players in our town could access the game. I am still hoping that it is possible to roll back to how the game was on Friday or Saturday when the problems started.
