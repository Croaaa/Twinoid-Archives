This is kind of a tedious thing to implement but I feel like it would really be worth it.

As you guys know, since Adobe will stop supporting Flash next year this game's playerbase will suffer a bit. Which is a shame.

I suggest remaking the world beyond exploration (and the explorable ruins) using HTML instead of Flash. This wouldn't be as fancy as Flash but it would not only allow Die2Nite to be played in any browser but also in Android and iOS. So that's a double win.

This would be implemented with clickable PNG images and simple scripts that would imitate the functionalities of the Flash interface, such as calculating the movement of players based on the tile they're at and storing tile information such as the number of zombies and the objects which happen to be on the ground.

In case the devs don't really care about doing all the work, I hope if the source code gets released some programmer stumbles upon this suggestion and does the code themself.

Also I apologize if this has already been suggested or if the devs have already talked about the future of this game. I just figured this was a good idea.
