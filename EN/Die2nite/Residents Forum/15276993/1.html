What's This About?We all have a rough idea on how "good" or "bad" a weapon is (how easily it breaks after one use) and we've gotten a good datasheet on that as well. The popular notion is that weapon A has an X chance of breaking after one use; therefore, a 1-X chance of not breaking.

This is a good guess of how the system works, but it may be a wrong one. At least, for Season 5, the way the game may be calculating breakage percentages could have possibly changed. There's a good chance the chance of breakage is NOT random. And I can prove it.


"Methodology"... I swear it's simple In my town before this one, The Uneducated Forests of Bacteria, I took out an MKII (loaded) and 2 spare batteries.

According to the wiki, the chances of the MKII of crushing a battery is ~28%; this means that we can expect each battery to last for 3.5 zombies (10.5 zombies with three batteries, so it's either 10 or 11 zeds).

The manner in which I used it to kill zombies, however, was slightly experimental. Let's label the three batteries (2 spares + 1 already loaded) as A, B, and C.

We start out with all three batteries in my rucksack. A was loaded first, then dropped to the ground. B was next, then dropped to the ground. C was last, and dropped to the ground. Now we have all three batteries on the ground.

What I did was I randomly chose one battery from the ground and loaded it onto the MKII, then picked up the other 2 batteries as well so that I have no idea which one is A, B, or C.

I rinsed and repeated the process. This made it nearly impossible for me to load the batteries in the same order (i.e., if I started with A-B-C on my first order, since I randomly picked batteries, I'll likely end up with some other order, B-A-C or C-B-A or A-C-B etc.)


"Results and Discussion"If the original notion of a breakage percentage was correct, then it shouldn't matter whichever order I use my batteries. Every 3 or 4 zeds I kill, I should be generating a crushed battery. In other words, if the original notion was correct, the number of times the batteries get crushed should be spaced out.

This is because it's a probability, a random chance. It doesn't have any memory of how many times it's been used up or how many batteries we have left in the rucksack. The chances of all batteries breaking are the same, so it shouldn't matter if we shoot them out of order. They will break with more or less reasonably large spaces between the batteries.

What I got, however, was this:

First Crushed Battery: 15 uses
Second Crushed Battery: 2 uses
Third Crushed Battery: 1 use

This suggests that each battery actually has a "memory" of how many times it's been fired before. In other words, the chances of breakage are NOT random!

Since I loaded rounds randomly, each battery had more or less been used an equal number of times before the first breakage. If they had a memory, then they should break at around the same time. Which they did.

If I'm right, further experiments should turn out the same way.


ImplicationsThere is some other mechanism at work for determining when a battery (or any weapon) breaks. And we don't know what it is. However, if we DID know, then there will be much more practical and specific ways to take advantage of that information.

My hypothesis is this: say we have weapon Y. The game generated weapon Y when you scavenged it. You then run into a horde of Z zombies. You use weapon Y to eliminate the first zombie.

What the game does is randomize a number, L, as the number of uses weapon Y must endure before it breaks. That is, right from the start, weapon Y already has L kills it can do.

If L > 1, then we know the first kill has a 100% chance of success.
If L > 2, then we know the second kill has a 100% chance of success.
And so on until the Lth kill, where the weapon still has a 100% chance of success.

But on the (L+1)th kill, the weapon has a 100% chance of breaking.

This is vastly different from our notion where the weapon has an X chance of breaking. If there was an X chance of breaking, we can have 1 kill or 100 kills, no limits, but each kill had an X chance of happening.


So What?If we can get more data on each weapon, we can determine the L for each weapon. Of course, that value fluctuates, so we can also determine an upper and lower bound for L.

This means that if you're pessimistic, you can safely use the lower bound for L and plan a zombie massacre knowing that you have enough weapons to kill at least as many zombies as you planned.

If you're optimistic, you can safely use the upper bound for L and know that no matter how lucky you are, you can never kill more zombies than the upper bound of L allows.

If I'm right and we have enough data, we can even map out a distribution for L. So we'll know exactly how lucky we have to be to reach a certain number of zombie kills inside the lower and upper bound for L.

Useful so that maybe the difference between a 0% chance and a 10% chance is 1 zombie kill, and even as a pessimist we think we can't possibly be so unlucky to fall outside that 10% window. Then we can save bringing the Pig along and free up 1 extra space in the rucksack.

A random number generator does not randomly change every number in the game lol. The only number that is randomly generated (and thus changes) is the number that decides if the weapon breaks or not. To use the old dice analogy:

Say someone is using a screwdriver and it has a 30% chance of breaking (making this up as an example). Then we can pretend the server rolls a die every time someone uses an item (since the die will generate a random number). The server see's what item it is using, looks up the breakage chance, and rolls the die and determines whether it breaks or not based on the result.

A specific example of what could go on behind the scenes:
1) A player uses a screwdriver to attack
2) Server sees this and prepares.
3) Server looks up item ID number of "screwdriver", it is item number 1119
4) Server looks up breakage chance of item 1119, it is 30%
5) Server rolls one 10 sided die, if it lands on 1,2, or 3 then the item breaks, if it lands on 4-10 then it doesn't, random number generator (or "die") generates a 7, item does not break
6) Steps 1-5 repeat until a 1,2, or 3 is randomly generated.


So as long as the player does not change weapons and only attacked with the screwdriver then the ONLY number that will ever change is the number generated in Step 5. The ID number will never change, the breakage chance will never change because the random number generator is not used for those two numbers.
