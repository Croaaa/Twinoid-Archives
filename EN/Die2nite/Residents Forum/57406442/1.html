Dear fellow die2nite survivors !

Using the Twinoid API, I've made a plugin that makes a snapshot of the map every day!
For now, it makes a screenshot every second hour, at 22.55 and at 23.25.
I think I will decrease the number of snapshots later on, because keeping so many snapshots makes no sense at all .

You can register your account by going to the registration page, and click the big button, which redirects you to Twinoid. After authorizing, your account will be added to the database, and your maps will be updated on set times automatically.
However, for now only my friends and API-validators can register, . Also, there is no way to un-register
From then on, the site will make a snapshot of the map that you are in. No double snapshots will be made when other people from your town have registered!

You can check out the application here: http://jasp.party .

I'd love to receive some feedback on the functionality or design. There are some things that I have on my mind as well.

Currently maps are shown by ID. As no one knows the ID of the map, I want to substitute it for the map name.
The citizen list is long and certainly not wide. I'm thinking of making it blocks that are after each other, so that you have multiple names on one line.
Snapshots are sorted on alphabetic order, which makes "day 1, 04:00" come after "day 1, 23:25". Another order will be better.

Please share your thoughts

Exact zombie counts can be extracted from the API, when the citizen registered to the API is standing on a certain zone. An update usually occurs when a player moves from one zone to another, which is done by a browser plugin (or manually). The API will give the exact number of zombies for that zone. This is stored and kept on the servers (cartographer, fdtd, map viewer, ...) For the unvisited zones, the estimation by Die2nite is used (the same as that I am using).

However, I am not requesting the API keys of all players, neither am I providing a browser plugin. This is also not the goal of the app, because I know that it will turn out like all map-apps except for FDTD: The data will be provided just by a few people, incomplete and potentially unreliable.

This makes it impossible for this application to make the zombie count more precise, just by using the Die2nite API. As FDTD does not provide an API of any sort, I cannot (legally, without scraping) extract the zombie count from FDTD.
