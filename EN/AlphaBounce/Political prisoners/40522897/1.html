Hi guys.

So I made this topic, because like some of you know, it's difficult to buy lots of if you have a lot of .

Today I finally arrived at [53][-124] and I had enough to buy 286 (can't buy anything else since I've bought everything ). So I started buying this item, but as you may well know, you can't specify that you want to buy X amount of some item and that means you have to repeat an action potentially hundreds of times.

To avoid this, I started a bit of research and I've came up with the following solution:
- found software AutoHotKey. It's free, open source, etc
- installed it and created on the desktop a new script (Right click -> New -> AutoHotKey script)
- named it Alphabounce.ahk (name is not important) and right clicked it, then 'Edit Script'
- added the following content to the script (the first 4 lines were already filled in, I left them intact):


#NoEnv ; Recommended for performance and compatibility with future AutoHotkey releases.
; #Warn ; Enable warnings to assist with detecting common errors.
SendMode Input ; Recommended for new scripts due to its superior speed and reliability.
SetWorkingDir %A_ScriptDir% ; Ensures a consistent starting directory.

IfWinExist AlphaBounce - Web game
WinActivate
Sleep, 300

Loop, 280
{
Click 480, 260
Sleep, 300
Click 680, 550
Sleep, 700
}

return


- saved the file, closed it
- open AlphaBounce in the browser, go to the merchant and open to see the buttons to choose the items and to buy them
- minimize the browser, double-click the script file to run it

Now to explain what the script does:
- it will look for a window with the title "AlphaBounce - Web game" (which is your browser, and it's minimized)
- it will activate that window (just like clicking on the browser in the taskbar)
- it will wait 300 milliseconds
- then it will repeat for the number of times you specify in the loop (in my case 280, buy you can try with a smaller number to get the hang of it and run the script multiple times. you can also edit the script later before running it again) the following instructions: click at the 480, 260 coordinates (in pixels) to select the item (this coordinate applies for my resolution, at the merchant I visited. Your mileage may vary - most likely - and you have to figure out the coordinate for the button you're interested in for your case). Then it will wait 300 milliseconds. then click at the 680, 550 coordinate, which in my case was the buy button, then wait 700 milliseconds then repeat the whole process.

I used those timeouts by trial and error. If you don't wait enough the game will hang as it can't process the requests that fast. Also, if you have a slower connection, you might need to increase those values. The values should be updated accordingly, based on your situation.

What this does is spare you hundreds of clicks and moving the mouse between the two buttons. It basically automates an action. Other than that, it's just like you doing it, there's no trick, no hidden actions.

Hope this tip helps anyone else who runs into this situation .

Cheers !
