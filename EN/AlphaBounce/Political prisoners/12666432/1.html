Most server systems are ready for situations like this. At least they should be.
One or the other query would reach server first. If this is undiscovered square server blocks those coordinates for a while(less than a few seconds) to generate a level.
The second will have to wait this little while and start the game after level is generated and coords are released. In worst case scenario, you'll get an error just to reconnect in a moment.
These are the basics And probably MT did this in more efficient way(and less simple).
