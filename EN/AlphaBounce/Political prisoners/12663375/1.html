It's an interesting (if straightforward) mathematical problem to determine what the steady-state cost of the auctions will be in a closed system. You need to make the following assumptions:

1. The total number of users will not increase. Call the total number of users A
2. The average number of gems collected per day is fixed. Call this number X
3. The total number of auctions happening in a day will not change. Call this number B.
4. People will bid up to their maximum number of gems to win an item (ie, each item will sell for the maximum number of gems possible)

It shouldn't be too hard to see that the steady state cost of the auctions will be:

AX/B

If the auction values are starting to stabilize now, I think what you can expect is a much slower increase in the prices. You should be able to start to catch up, as what's happening is that the fast gem earners have almost all made their first purchase, and have to start earning again from zero gems.
