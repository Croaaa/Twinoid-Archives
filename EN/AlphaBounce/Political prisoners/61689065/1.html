There is another unintended mechanic with the R powerup that involves creating a ball while out of bounds. The game will not allow you to do so, but will still update the "ball-counter" making it one higher. How the game actually checks to see if the envelope is destroyed is not by counting the number of balls on the screen - it starts it at 1 (2 if FURI) and then manually tracks it as new balls are added or removed. When you attempt to create the ball out of bounds it will add one to your "on screen ball count" without actually creating the ball. The result is that that counter can never reach 0 (which is when it destroys your envelope) because there is no way to destroy the ball that was never created.

This is not particularly useful using the normal control scheme, but can be done very easily with keyboard controls.

This is not an automatic clear, as your envelope may still be destroyed by guardians, kashuats, orbital nuts, etc. which will reset the counter to 1 (2 if FURI).
