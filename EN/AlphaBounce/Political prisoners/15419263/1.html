Made another small update to the Overview Map.

Because I use the beacon pretty often to mark my position on the map (for orientation purposes), it was pretty annoying when I tried clicking my exact coordinates (because sometimes it's hard to move the mouse exactly the number of pixels you want), so I added the possibility to manually edit the position of the beacon.

In order to do this, just click on the beacon coordinates on the top left of the page (even if at first load they're not set and it only shows "[ ][ ]") and two text inputs should appear. As you guessed, they're for the X and Y coordinates and they will show the current coordinates. After you edit the values, you need to press Enter (works in any of the two fields) to update the beacon position. If you change your mind, you can press Escape and the old coordinates of the beacon remain. If you enter some invalid values, when you press Enter, nothing happens.

The new version is 1.13.0 (you can see it under the "Alphabounce map" title. If you see anything lower than that, you probably need to clear your cache.

As always, I recommend using Google Chrome (it loads faster, and it's smarter about numeric input types and it will show small arrows for increasing/decreasing the values), but Firefox will work as well. I can't vouch for any other browser (*cough* IE *cough*).

Hope this little feature is useful for anyone else except me. If not, I did some work just for satisfying a little whim of mine .

Hmm... You are right, invisible area allows full jumps on diagonals and lower jumps by axis. Let's see... Formula for engine (known area) is correct. For unknown area shortest path will be:
D+S, where D - quantity of possible diagonal moves, S - quantinty of axis moves you have to make.
D=Math.ceil(Math.min(abs(x1-x2),abs(y1-y2))/Dp), where Dp - number of units you can move on any axis when jumping diagonally.
S=Math.ceil(abs(abs(x1-x2)-abs(y1-y2))/Ap), where Ap - number of units you can move by axis.

If you want to clear the map, press the "X" button from the section. If you click the "Editor" button, it will show/hide the editor. At first it's empty. When the editor is empty, if you click one route, it will load it's content in the editor (I'll explain the structure in a moment). Normally when following a route, you should load it in the editor then keep a cursor at the current line or something. I haven't though yet of implementing something to keep track of your current position, also nothing is persisted about this feature for now.

Now to explain the type of lines that you can encounter in a route (each route entry starts with a code and can have zero or more values, separated by commas):

- lines starting with "sn" are for section names, purely informative (second value is the section name)

- lines starting with "o" (letter o from one, not number zero) are for specifying the coordinates of the origin from where the route starts. This is not mandatory. If not provided, the origin is the first move. Values are x, y and optional number of minerals (maybe I should remove this, doesn't seem that important).

- lines starting with "m" are for moves. Same as "o", it shows the coordinates to which the move should be. Has x, y, number of minerals (minerals including drones maybe... up to us, it's just informative). For now nothing is done with the number of minerals, but who knows, maybe we can use it in the future. I expect this type of entry to make up the bulk of a route.

- lines starting with "d" are for death levels... this is when you need to move to a level to get a bunch of minerals, but you need to lose it so you don't spend too many . Same values, x, y, minerals (for this, the map should continue the line properly, but it's not implemented yet)

- lines starting with "c" are for changing the color of the line. The default is #fff608 but as everything, open for suggestions. The second value needs to be the haxa code of the new color. maybe this can be used for different portions or something... not sure what it's use could be, but it might prove useful.

- lines starting with "u" signify that it's an unknown sector and the user should search for a level that has more minerals. This can usually be between two points where you don't have good alternatives so the users can find out new rich levels (and maybe share back with the others so the routes could be improved). This line has no coordinates, it should just change colours (uncharted color code is #ff5408 for now). When the next "m" entry is encountered, the color switches back to whatever was used before "u" (either the default or a color that's been changed with "u".

These are the type of commands I though of, but if you have other ideas, maybe we can improve this.

It has no backend functionality, so feel free to save it and host it yourself if you want/need. I use it on my phone (I found it works better with Chrome, as Firefox kind of zooms in when you want to enter some numbers). Basically you choose the number of players, enter the names, enter the number of rounds if you know you want to have a fixed number of games and see who has the biggest score after that (we do that with some simple darts game for example, we play 5 rounds) - if you don't want that, leave the default infinite and it will create new rounds as you start typing scores. Check the 'Show dealer' option if you want to see a highligh for each round of who is dealing the cards or whatever it is you're playing. After starting the score, just fill in the numbers and that's it. It will show in blue the total score, it will show the difference between all players and the first player and the ranking. If you want to change who the dealer is, there's a menu that can be shown for each cell, you get it by double-clicking or double tapping on mobile. The "D" button makes the current cell the dealer, the "*" button is for rummy, if you have the card that's on the deck (not sure how that's called, but you get 50 points extra for it) and "-/+" if you want to change the sign of the score (on some mobiles, for the numeric keyboard you couldn't type -100 for example, so you could write 100, show the menu and change the sign). On the top-left corner there are two buttons "R" to reset and choose a different number of players and names or "A" to play again with the same players, just resets the rounds and scores.

That's it, hope it helps anyone .
