I'm not a flash expert either, but I do write several programming languages. What you were seeing was most likely an array missing a [0] value. It was probably the result of the Multiball powerup.

A little background: Most arrays begin at 0 and count up for each value. Some arrays store one value, others can store miltiple values. If the programmers used an array to display the balls on screen, the array might look like this with 1 ball:

[0] = 323,938

When you get a multiball, it might look like this:

[0] = 323,938
[1] = 230,302

However, because you got the multiball just as your remaining ball went off screen, the program probably called an "array insert" function at the same time the "array remove" function was called but before the "ball lost" function was called. So the array ended up looking like this:

[0] = null,null
[1] = 230,302

So, when program called the array to decide where the ball was (you got a power up) it hit the "null" values, the program realized the array was invalid. So, an error check decided that the "null" values should be equal to "0", meaning the game read the array as:

[0] = 0,0
error

It didn't process the rest of the array because the first entry was invalid. The 0,0 coordinate would be in the upper left (which is exactly the same as the global map you travel on) and therefore spawned a non-moving volt ball in the upper left hand corner. Without the usual mathematical functions to make the ball move, the ball simply sat there doing nothing.
