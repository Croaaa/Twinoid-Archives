If we are going to make this happen then I would suggest the following format:

There is a list of links to armies that want recruits.
Each link has beside it ONE combination of starting kit that is desired.
There may be a maximum of three different requests for one army in the list at any time.

A player wishing to join the list copies and pastes it into their post, adding their request to the bottom.

A player wishing to help should start at the top of the list and work down, checking each link to see if a suitable recruit is available.
If a suitable recruit is made then that request should be deleted and the list re-posted.
(a player fulfilling a request may choose to keep score of how helpful they have been... that's the only reward they'll get! Unless we want to say that you can only add yourself onto the list if you meet a request... but I don't think we need that)

Managing requests for passwords could be tricky, as then you'd have to keep track of who made each request too.
Maybe adopt a similar format to the Championship Sixes thread?

Example list:

http://linky.link.minitoopers.com/ owner-twinoid-name / shotgun + sprinter
http://someone.else.minitroopers.com/ other-person / pistol + smart + password

If someone makes a recruit for someone.else army then they can send the password to other-person via twinoid pm.

Does that sound workable? If so then someone could make a new take-a-penny-leave-a-penny thread and get things going.
If it's not quite workable then what needs fixing?
(of course, an automated version of this would be much less clunky)
